## Fixed
- **Anycubic detail: the Files button is back in its proper place.** It floated alone in a head row above the camera/controls; it now sits inside the Control card next to the light (a `[folder][light]` row), exactly like Snapmaker — same shared `.elg-ctrl-actions` layout. `renderer/printers/anycubic/index.js`, `renderer/printers/anycubic/cards.js`
- **Sidebar version text keeps the same size collapsed or expanded.** It shrank from 10px to 9px when the sidebar collapsed; the collapsed override is gone so `v1.10.26` stays 10px in both states (it still fits the narrow rail). `css/00-base.css`
- **Sidebar GitHub / 3D Files / Discord buttons are now the same height expanded and collapsed.** Expanded they had no fixed height (it came from padding + content ≈32–34px); collapsed they were a fixed 34px — so they resized slightly when the sidebar toggled. A shared `height: 34px` now applies in both states (collapsed only changes their width). `css/10-settings.css`, `css/00-base.css`
- **Loading buttons no longer show an icon AND a spinner at once.** A button in its `.loading` state hid its label and showed the spinner, but kept its leading icon — so e.g. the Anycubic cloud "Connect" button showed a refresh icon next to the spinner. The icon is now hidden during loading too (generic `button.loading .icon`), so the spinner sits alone. `css/70-detail-misc.css`
- **Sidebar avatar no longer jumps right-then-left when you collapse the menu.** The collapsed state centred the avatar with `justify-content: center` on its row, applied the instant the `.collapsed` class lands — while the sidebar is still wide, so the avatar snapped to the centre of the wide rail, then slid back left as the width animated. Now the avatar is centred purely by padding (`(65−40)/2 = 12.5px`, flex-start), so it stays put and the sidebar just shrinks around it — collapse mirrors expand. The friend chips had the same problem (`.sb-friends-list` was `align-items: center`, and the list was content-sized so `.sb-actions` re-centred it too): the list is now `width:100%` + `align-items: flex-start`, so the chips also stay left and don't jump on collapse. `css/00-base.css`
- **Sidebar avatar no longer dips down-and-back when you expand the menu.** The "welcome" line next to the avatar lacked `white-space: nowrap`, so while the sidebar was still narrow mid-expand the text wrapped to extra lines, briefly making the avatar's row taller (the centred avatar dropped, then sprang back as the text un-wrapped) — a visible jump. The welcome line is now single-line (nowrap + ellipsis) so the row height stays constant through the animation. `css/00-base.css`
- **Sidebar collapse/expand no longer makes the Notifications row flash, and the collapsed bell is now centred.** The button's label used a hard `display:none` in collapsed mode, so it popped in/out instantly while the sidebar width animated — a visible jump. It now animates shut (label max-width + opacity, with a synced button-width transition) so the menu glides open/closed cleanly. The Notifications button also lacked `display:flex`/`align-items:center` (unlike the community buttons), so the bell sat off-centre — now flex-centred. The label needed `min-width:0` too: as a flex item its default `min-width:auto` kept the text's width (invisible but space-occupying), shoving the bell left even when collapsed. The collapsed button is now 40×40 (was 36px) to match the friend avatars so the column lines up, and the bell glyph is a consistent 20px in BOTH expanded and collapsed states (was 13px) so it has presence and doesn't resize when the sidebar toggles. Final centring is done ABSOLUTELY (`position:absolute; left/top:50%; translate(-50%,-50%)`) on the collapsed bell so neither the 0-width label nor the absolutely-positioned notification badge (children the community buttons don't have) can shove it off-centre. `css/00-base.css`
- **Switching view now always closes any open spool/group card.** Unified all view transitions (account↔account, your inventory→friend, friend→friend, friend→your inventory) through one `_closeAllSideCards()` helper. Before, only entering a friend from your own account cleared the cards — going friend→friend left the previous friend's spool/group panel open, so you'd see someone else's spool inside the next person's view. `renderer/inventory.js`
