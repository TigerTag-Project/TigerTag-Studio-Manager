# Worklog — v2.9.0

## Added
- **TigerPOD-owner census — dual signal (declared + auto-detected):** `hasPod` (toggle OU auto au 1er scan) + `rfidReadersMax` (1|2). A new "I own a TigerPOD" toggle in the TigerPOD modal writes `users/{uid}/telemetry/studio.hasPod`; it's also auto-set on the first successful RFID read. Seeded at login (`hydratePodSignal`) so the toggle reflects reality. The RFID badge now opens the TigerPOD modal whether a reader is connected or not — `renderer/inventory.js`, `renderer/inventory.html`, `renderer/css/60-modals.css`, `docs/firestore-schema.md`.
- **The "To order" cart is auto-grouped by purchase source.** The active cart splits into one group per buy-link host (a sticky header labelled "atome3d.com", "amazon.fr", …, with a per-group count), and everything without a link falls into a "No buy link" group shown last. Display-only grouping — one drag zone (headers ignored by the make-room reorder), but **a drag-reorder is now constrained to its own group**: an item can only be re-ordered among items sharing the same buy source, or within the saved-for-later shelf — never dragged across groups or between cart↔saved (the move-to-saved/back is done via the line button, not drag). The payment total sums across all groups. Saved-for-later stays one shelf — `renderer/inventory.js` (`_renderOrderTab`), `renderer/css/70-detail-misc.css`. New `reorderNoLink` i18n (9 locales).
- **Per-item quantities in wishlists (Amazon-cart style).** Each list item gains an **Amazon-style quantity selector** (owner) / read-only `× N` (friend): a dropdown 1–9 + "10+" (all 10 rows shown at once, left-aligned, selected row highlighted — like Amazon), where picking "10+" swaps to a hand-typed number input (no spinner arrows). The dropdown is a body-appended `#qselPop` positioned in JS (`position: fixed`), so it overlays everything and never clips inside / grows the card; it closes on outside-click, scroll or resize. Shared `_qtySelHTML` / `_commitQty` widget + `_qselOpen`/`_qselClose`, also used by the "To order" cart. Quantities are stored in a new `itemQty` map (`keyHash → n`, default 1) on the list doc — deep-merged so items don't clobber each other. The payment-summary total and article count now multiply by quantity, and the public snapshot (`publicLists/{token}.items[].qty`) carries it for the web page. Backend: added `itemQty` to the `lists` `hasOnly()` whitelist (deployed). New `_listQtyOf` / `_setListItemQty` helpers, `data-listqty` wiring, `.lv-qty` stepper — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`, backend `firestore.rules`.
- **Payment-summary card in the Lists view** — a right-rail card showing the list's amount (Subtotal · N articles, estimated VAT, Total), identical to the "To order" cart's payment card (reuses the `.pv-summary` markup + `pvPayment`/`pvSubtotal`/`pvItems`/`pvTax`/`reorderVat`/`pvOrderTotal` i18n). Total = Σ `buyPriceHt` over the list's priced items (one unit each), shown in the account's HT/TTC mode + currency; only rendered when at least one item has a price. Stacked above the public-share card in a new `.lv-rail` right column (works in friend-view too) — `renderer/inventory.js` (`renderListsView`), `renderer/css/70-detail-misc.css`.
- **QR codes are now generated locally (offline) — no external service.** Replaced every `api.qrserver.com` call (public-list card + downloadable PNG, friend-invite card, the two onboarding QR cards) with local generation via the vendored `qrcode-generator` (MIT, `renderer/lib/qrcode/qrcode.js`, loaded before `inventory.js`). New `_makeQrDataUrl()` renders the module matrix to a canvas and returns a PNG data URL — works with no network and no third-party dependency.
- **Customisable QR style** — the "Public link" card (owner only) gains a colour-swatch row (5 presets); the choice is stored per device (`localStorage tigertag.qrStyle`) and applied to all of your shareable QRs (public list **and** friend invite) — `renderer/inventory.js` (`_qrStyle` / `_makeQrDataUrl`), `renderer/css/70-detail-misc.css`. New `qrCustomizeLabel` i18n (9 locales). (A centre-logo toggle — a tiger-head mark from the contouring logo, `assets/svg/logos/logo_tigertag_head.svg`, drawn on a white backing, ECC bumped to H — is built but **disabled for now**: the toggle is hidden and `_qrStyle().logo` is forced `false`; the rendering code + asset are kept for later re-enable.)
- **Download button on the public-list QR code** — a square download button sits to the right of the "Copy link" button on the "Public link" card (Lists view) and saves the QR as a crisp high-res PNG via a native Save dialog. The `image:download` IPC now also accepts a `data:` URL (the locally-generated QR — decoded + written directly), alongside the existing remote-URL path — `main.js`, `preload.js`, `renderer/inventory.js`, `renderer/css/70-detail-misc.css`. New `listShareDownloadQr` i18n (aria-label, 9 locales).
- **Apple/Bambu-style version carousel in the "What's New" modal** — a page-control of round dots (one per version) in the footer where the **active dot is an elongated orange capsule kept permanently centred**: the whole track slides under it (translateX, animated) so you can go back and forth without the pill ever drifting to an edge, at any position including the first/last version. Fixed-width viewport (7 slots) with faded edges (mask gradient); each dot is a big invisible hit target (20×30px) with a small round `::before` so it's easy to click while staying clean. Click a dot to jump; the title-bar picker still jumps anywhere; browsing doesn't change the acknowledged ("seen") version — `renderer/inventory.html`, `renderer/inventory.js` (`_wnRenderDots` / `_wnGoTo`), `renderer/css/60-modals.css`.

## Fixed
- **False "low stock" notification fired at launch even when the spool was in stock.** `_checkLowStockNotifs()` runs on every `renderInventory()`, including the initial loading pass — before the first inventory snapshot arrives `state.rows` is empty, so every product with a minimum was counted as 0 in stock and a persisted "low stock 0/min" notification was pushed (staying visible after the real 1/1 stock loaded, since notifications aren't deleted). Now the check bails while `state.invLoading` is true or `state.inventory` is null, so it only evaluates once real inventory data is present — `renderer/inventory.js` (`_checkLowStockNotifs`).
- **Friend / follower count didn't update live when someone added you.** `_renderFriendsEverywhere()` (run on every friends-list change) repainted the friends list + badge but never re-rendered the header banner, where the count lives — so gaining a friend/follower (e.g. auto-accept on a public account) left the number stale until an unrelated repaint. Both sides were affected. Now `_renderFriendsEverywhere()` also calls `renderFriendBanner()` (cheap — signature-guarded), so the count updates immediately; it's still published to `userProfiles.friendsCount` in the same subscription pass — `renderer/inventory.js`.

Dropped from `_productBadgesHTML`; removed the now-unused `.prod-badge--shop` CSS and the `reorderHasLink` i18n key (9 locales) — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`.
