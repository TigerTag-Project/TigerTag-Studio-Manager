# Worklog — v1.10.10 (in progress)

## Added
- FlashForge **Printer info modal** — an "ⓘ" button on the job card opens a modal with lifetime stats (filament used in m, total print time, free disk space) and machine specs (model, firmware, nozzle Ø + count, build volume, group, IP, MAC, lidar, TVOC, auto-shutdown, Z offset), parsed from `/detail`. `printers/flashforge/index.js`, `printers/flashforge/cards.js`, `inventory.html`, `inventory.js`, `css/50-snapmaker.css`
- FlashForge live block now surfaces more of the `/detail` payload: (1) **temperatures Snapmaker-style** — one labelled pill per nozzle (tool-changer → `T1…Tn` with the active tool highlighted; single-nozzle → `E1`) plus BED and CASE, each `current/target°C`, all in one wrapping row. Per-tool temps moved off the filament cards into the Temperature block; (2) **fans** — a status strip ABOVE the temperature card (like the Bambu control card): Fan + Chamber (each shown only when its speed is reported) as Elegoo-style columns (icon+label / `− value% +`, the −/+ disabled since FlashForge has no fan-control command); (3) **safety** — a door open/closed chip (shown only when `doorStatus` is reported) and a red **error banner** when `errorCode` is non-empty. `printers/flashforge/index.js`, `printers/flashforge/cards.js`, `css/50-snapmaker.css`
- FlashForge machine controls + on-board file management (HTTP, all brands' models). (1) **Chamber/case light toggle** on the job card (`lightControl_cmd`, state from `detail.lightStatus`, optimistic). (2) **Pause / resume / stop** buttons while a job is active (`jobCtl_cmd` pause/continue/cancel). (3) **File explorer** — a folder button on the job card opens a bottom-sheet (like Creality) listing the printer's on-board gcode/3mf files (`/gcodeList`) with thumbnails (`/gcodeThumb` → base64 PNG) and a per-file **Print** button (`/printGcode` with the slicer's default options). Extended the `ffg:http-post` allowlist to the three file endpoints. Captured + verified against a real Creator 5 Pro. `main.js`, `printers/flashforge/index.js`, `printers/flashforge/cards.js`, `inventory.js`, `css/50-snapmaker.css`
- FlashForge Creator 5 / 5 Pro filament colour now uses the printer's **imposed palette** instead of a free colour picker. These models' CFS firmware only accepts its 24 built-in "Color Library" colours — any other value is silently rejected and the slot reverts to white. The slot-edit colour grid now shows ONLY those 24 swatches (no custom/free picker) for `printerModelId` 5/6, so every pick is one the printer will actually keep; other models keep the free picker. Palette (`FFG_CFS_PALETTE`) captured from a real Creator 5 Pro. `printers/flashforge/index.js`

## Changed
- Bambu H2-series **dual nozzle** now displayed. The H2C/H2D/X2D have two heads (`device.extruder.info[]` — id 0 = right, id 1 = left); the temperature card showed only the active one. It now renders **one pill per nozzle**, tagged **R / L**, with the active head highlighted, each editable independently (a dual-head setpoint targets its head via `M104 T{id}`; single-nozzle models keep plain `M104`). `printers/bambulab/index.js`, `printers/bambulab/cards.js`, `inventory.js`, `css/50-snapmaker.css`, `printers/bambulab/PROTOCOL.md` (§16.8)

## Fixed
- Filament edit bottom-sheet was hidden behind the printer side-card. The 3-panel z-index ladder (101-106) raised the printer panel above the edit sheet (`.sfe-sheet` was 103-104), so opening a slot's colour/material editor showed nothing — it opened behind the panel. Raised the sheet stack above the ladder: `.sfe-backdrop` 110, `.sfe-sheet` 111, sub-pickers (`--color`/`--filament`) 112. Affects all brands' filament edit sheets (Snapmaker/Creality/Elegoo/FlashForge). `css/50-snapmaker.css`
