## Changed
- **"NOT STORED" stat tile only goes orange when there's a backlog.** At count 0 it now renders neutral like the other stat tiles (RACKS/SLOTS/‚Ä¶) instead of always-orange. `renderer/inventory.js`
- **Friend view always opens on the Materials grid.** Entering a friend's inventory ‚Äî or switching from one friend to another ‚Äî now forces the Grid (materials) view instead of keeping a stale Storage/printer/table view. Done transiently (`setViewMode("grid", { persist:false })`) so it doesn't overwrite the owner's own saved view, which is restored when switching back to their account. `renderer/inventory.js`
- **View-toggle groups are now labelled.** The two otherwise-identical Grid/Table segmented controls each carry a short icon+label prefix ‚Äî "Materials" (Grid/Table/Storage) and "Printers" (Grid/Table/Cam) ‚Äî so it's clear which one switches the material inventory vs the printers. `renderer/inventory.html`, `css/70-detail-misc.css`
- **"Spools not stored" is now a permanent, always-visible docked column** instead of a slide-in panel. The Storage view no longer scrolls as a whole: the header stays put, the racks scroll in their own area (`.rp-racks-scroll`), and the panel is a full-height column on the right that's ALWAYS shown ‚Äî even when empty, where it renders a clear dashed "drop a spool here to un-rack it" zone so new users know where unstored spools go. Its list scrolls inside when there are many. Removed all the open/close machinery (chevron handle, click-outside-to-close, auto-open animation, drag hide/show) and the `‚úï`; the "NOT STORED" stat tile is now an informational counter (no toggle). Dropping a rack spool onto the column un-racks it (existing `#rpUnrackedStrip` drop target). While a spool is being dragged, the empty drop zone lights up WeTransfer-style (primary dashed border + tint + pulsing glow) so it's obvious it accepts the drop; on direct hover it goes steady (no double dashed outline ‚Äî the list's own drop outline is suppressed when empty via `:has()`). The empty state uses a download icon + an action title ("Drop a spool here") + an explanatory hint so its purpose is clear. When dragging a spool OUT OF A RACK and the column already holds spools, the rows hide and reveal that same drop-zone message, giving a clear free area to drop onto (rows reappear on drag end). This is gated on `is-dragging-racked` (not any drag) so grabbing a row FROM the column to place it in a rack keeps the rows visible ‚Äî otherwise the drag source got yanked and couldn't be placed. Racks re-pack automatically via the masonry width observer. `renderer/inventory.js`, `css/30-racks.css`
- **Spool detail card max width capped at 350px** (instead of 85vw) ‚Äî `makePanelResizable` got an optional `maxW` param; saved wider widths are re-clamped on load. `renderer/inventory.js`
- **Printer settings gear button now toggles the panel** ‚Äî clicking it again while the settings panel is open closes it (was open-only). `renderer/inventory.js`

## Fixed
- **Auto Unstorage: empty spools can no longer be dragged out of "Spools not stored".** A 0g spool can't be stored while Auto Unstorage is ON (it'd be freed and bounced straight back), so it's now locked: non-draggable, dimmed with a lock badge, and a hover tooltip explains why (`rackUnstoreLockedTip`). Flipping the toggle re-renders immediately so empty spools already in the bin lock/unlock right away. `renderer/inventory.js`, `css/30-racks.css`
- **Auto Storage: "Clear all" hidden + dropping in the bin opts out.** Clearing a rack would just send everything back to the pile for Auto Storage to re-scatter, so the kebab "Clear all" item is omitted while Auto Storage is ON. The "not stored" bin still accepts a drop while Auto Storage is on, but doing so is treated as an explicit opt-out: it **turns Auto Storage OFF** (toggle + `body.autostore-on` updated) and then un-racks the spool, so it actually stays in the bin instead of bouncing back. During such a drag the bin swaps its hint for a note explaining the side effect (`rackAutoStoreNoDrop` ‚Äî "Drop here to keep it unstored (turns off Auto storage)"). Rack *deletion* still frees its spools (which Auto Storage re-places automatically while it's on). `renderer/inventory.js`, `css/30-racks.css`
