# Worklog — v2.14.1 (in progress)

## Added

## Changed
- **The reorder card's "Add a minimum" (minimum-stock) control now matches the "Add a buy link" / "Add a price" buttons.** It was a read-only text field + an always-visible pencil; it's now the same full-width `.ro-shop-btn` (a bell icon + "Add a minimum" when empty — click to add; the value when set, with the pencil appearing only once a minimum is set, and the button turning green/`--active`). Clicking the button anywhere opens the inline editor, like the price button. Dropped the now-unused `.ro-price-shown` / `.ro-price-shown--empty` CSS — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`.

## Fixed
- **Spool/product thumbnails can no longer be "picked up" and dragged as a ghost image** in the inventory grid/list or Favorites, where no drag is intended. Browser `<img>` elements are native drag sources by default, so a thumbnail could be grabbed and dragged around with no drop — looking broken. Disabled it globally (`img { -webkit-user-drag: none }`); every intended drag (rack slots, list/cart reorder, friends) uses a draggable `<div>`/`<span>` handle rather than an `<img>`, so nothing else is affected — `renderer/css/00-base.css`.
- **Rack view: the whole panel no longer lights up amber while dragging a spool from one rack to another.** Dragging a spool over any empty area of the rack view (the gaps between racks, the edges of the zone) added `rp-view--drop-void` to `#invRackView`, whose `box-shadow: inset 0 0 0 2px rgba(234,179,8,.35)` glowed around the entire panel — a "drop here to un-rack" cue that fired constantly during a normal rack-to-rack move (which passes over empty space), so the whole card appeared to illuminate near the edge. Removed the glow. The void-drop **function** (release a spool on empty space → un-rack it) is unchanged; only its distracting full-view highlight is gone (un-racking also stays available via the unranked side panel, which keeps its own drop outline, and the toolbox "remove from rack" button) — `renderer/css/40-printers.css`.
- **Rack view: dragging a spool no longer lights the whole rack card orange.** The ambient "drop-zone" treatment painted every empty unlocked slot with an orange dashed border + soft-orange background + a `rp-slot-pulse` animation, so a card full of empty slots read as one big orange glow. Removed it: empty slots now stay neutral during a drag and only the slot **directly under the cursor** (`.rp-slot--drop`, unchanged) lights up orange. Locked slots still dim to read as forbidden. Also dropped the dead `body.is-dragging-spool .rp-rack { transition: box-shadow }` (no box-shadow was ever applied to the card) — `renderer/css/30-racks.css`.
- **The `.ttag` import overlay no longer pops up mid-drag when moving a spool in Grid/Rack view (Windows).** The full-window drop hint keyed off `dataTransfer.types.includes("Files")` to tell a real file drag from the storage view's internal spool DnD — but on Windows/Chromium, dragging an element that contains an `<img>` (a spool thumbnail) leaks `"Files"` into the drag types, so an internal drag was mistaken for a dropped file (random, depending on whether that card's product image was loaded; never reproduced on macOS). Added a `dragstart`/`dragend` guard: an OS file drag never fires `dragstart` inside the document, so any active internal drag now suppresses the overlay reliably — `renderer/inventory.js` (`_wireTtagDropZone`).
