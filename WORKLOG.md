# Worklog — v1.10.28 (in progress)

## Added
- Tag filter — a new dropdown (next to brand/material/type) listing every tag that exists in the inventory; selecting one keeps only spools carrying that tag. Wired into all filter paths (filteredRows + keystroke filter + storage-view dim), populated from `_allTags`, reset by clear-all — `renderer/inventory.html`, `renderer/inventory.js` (`populateTagFilter`, `filteredRows`, `applyInventoryFilter`, `applyRackSearchDim`)
- Grid view sort control — the grid had no sort UI (it only inherited the table's column-header sort). Added a sort dropdown + asc/desc toggle (shown only in grid view, same `state.sortCol`/`sortDir` as the table headers; columns: brand/material/name/type/weight/capacity/updated) — `renderer/inventory.html`, `renderer/inventory.js` (`_syncGridSort`), `renderer/css/70-detail-misc.css`
- Tags / Balises (Phase 1) — free-form per-spool labels (Shopify-style), stored as a `tags` string[] on the inventory doc. Editor in the detail panel: chips with ✕ to remove + an add input with autocomplete from existing tags; normalised (trim, ≤32 chars, case-insensitive dedup, ≤20/spool). Writes are twin-mirrored (both chips of one physical spool share tags). Tags are pure Studio metadata — never written to the chip, no re-burn. Searchable everywhere (added to the shared `_rowMatchesSearch`). New: `tags` in `normalizeRow`, helpers `_normalizeTag`/`_allTags`/`_writeSpoolTags`/`addSpoolTag`/`removeSpoolTag`, detail-panel section + wiring — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`. Schema documented — `docs/firestore-schema.md`. (Phase 2: chips on cards + multi-tag filter; Phase 3: saved "smart groups" by condition.)
- RFID physical-chip list + TigerTag+ backup — new `users/{uid}/rfidList/{UID_HEX}` subcollection (doc id = hex UID, auto-dedup on re-use). Counts unique physical chips, stamps `firstSeenAt`/`lastSeenAt`/`seenCount` once, and stores a tag+ signature `backup` (raw pages 0x04-0x27) the first time a plus chip is physically read — the backup's mere PRESENCE marks the chip as a TigerTag+, so no type field is stored. Census runs once per account on first live inventory snapshot (pushes every existing physical UID silently); backup captured on auto-scan. NO in-memory index: two plain boolean markers on the inventory doc itself are the dedup signals — `rfidListed` (UID has a rfidList entry) + `rfidBackup` (tag+ signature stored) — read straight from the already-loaded inventory, so a fully-synced chip costs ZERO rfidList reads/writes. Markers are wiped only when the inventory doc is deleted; a delete-then-rescan re-enters unmarked and the scan path verifies the rfidList entry doesn't already exist before (re)creating, so firstSeenAt/backup are never overwritten. New helpers `censusRfidListFromInventory` / `recordRfidChipScan` — `renderer/inventory.js`. Raw pages now attached to the auto-scan payload (`_rawPagesHex`) — `main.js`. Owner-only Firestore rule `users/{uid}/rfidList/{chipUid}` + `rfidChipsTotal` telemetry counter — `TigerTag_Firebase_Backend/firestore.rules` (separate repo). Schema + cross-client sync contract documented for mobile alignment — `docs/firestore-schema.md`.

## Changed
- Aligned `docs/firestore-schema.md` with the actual model (firestore.rules): fixed `inventory.twin_uid` → `twin_tag_uid` (was wrong), added `inventory.updatedAt` + `users.apiKey6`, completed `telemetry/studio` (all 33 whitelisted fields), and documented the 6 previously-missing subcollections (apiKeys, printers/{brand}/devices + secrets, notifications, racks, scales, uidMigrationMap) — `docs/firestore-schema.md`
- Debug → Firestore explorer quick chips — added a `rfidList` chip (lists the saved physical-chip UIDs + their data) and one chip per printer brand that auto-fills its per-brand device path `users/{uid}/printers/{brand}/devices` (the bare `printers` collection only holds empty brand docs) — `renderer/inventory.js` (`fseInit`, `FSE_QUICK`)
- Debug → Firestore explorer now fetches every doc of a collection (removed the 20-doc cap) — `renderer/inventory.js` (`fseFetch`)
- Firebase Explorer: the `users/{uid}/printers` path now shows a virtual brand-folder view (the brand parents are fieldless so a normal query + the client SDK can't list them) — probes each known brand's `devices` for its count, brands in use stand out, empty ones dim, each row opens that brand's devices — `renderer/inventory.js` (`fsePrinterFolders`), `renderer/css/70-detail-misc.css`
- Removed the dead `tags` quick chip from the Firestore explorer (no such collection — no code, schema or rule) — `renderer/inventory.js`
- Firebase Explorer is now its own dedicated, exploration-optimised side card (`#fseExplorerPanel`, opened by the Settings → Tools button) instead of a tab inside the Debug panel. Rich browsing UI: toolbar with a scrollable clickable breadcrumb + refresh/copy; documents shown as a clean key→value table (Firestore timestamps formatted as dates, booleans as badges, numbers/strings colour-typed, long strings truncated, nested objects as JSON); collections shown as collapsible rows (chevron peeks the fields inline, the id drills into the doc) with a one-line field preview, a type badge + doc count, and a live id filter; loading/empty/error states. A toolbar `{ }` button opens the raw JSON in a THIRD side card (`#fseRawPanel`) that live-mirrors the current result. Both cards wear a forced dark dev-tool theme (local CSS-var override → dark surfaces, light text, slim dark scrollbars, accented titles) regardless of the app theme. Opening the Explorer closes the Settings card it launched from. All cards resize, stack right-to-left with a proper z-index ladder (Settings → Explorer → Raw, each tucking behind the one to its right), and are mutually exclusive with the now API-only Debug panel — `renderer/inventory.html`, `renderer/inventory.js`, `renderer/css/70-detail-misc.css` (`openFsExplorer`, `openFsRaw`, `fseFetch`, `_fseRenderFields`, `_fseFmtValue`)

## Fixed
- Notifications center side card: opening it now closes every other open side card first (detail / group / printer / settings / debug / explorer / container + Friends), and it no longer dims/blocks the rest of the UI — it opens without a backdrop (like the spool-detail / printer side cards), so the interface stays clickable while it's open; closes via the ✕ or its new `»` close tab — `renderer/inventory.html`, `renderer/inventory.js` (`openNotifs`, `closeNotifs`)
- "Locate spool" (detail panel → Storage view) no longer dumps the UID into the search bar (which also left a clear-× that never showed). It now lights up just that spool's slot via a transient highlight (`state.locateUid`), and the next click anywhere clears it back to the normal unfiltered view — `renderer/inventory.js` (`btnLocateSpool`, `_startLocateHighlight`, `applyRackSearchDim`)
- RFID chip census never ran for users with a cached inventory — it was gated on the first snapshot AND `!fromCache`, but a cached user's first snapshot is `fromCache` and the following server snapshot is metadata-only (early-returned). Now triggers on the first SERVER snapshot (idempotent via the internal once-guard) so the whole existing inventory gets listed — `renderer/inventory.js` (`subscribeInventory`)
