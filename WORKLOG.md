## Changed
- **The login modal never opens by itself any more** — it is reached only by an explicit user action. The four automatic openings were dropped: cold start with no saved account, the active account's session expiring, and both sign-out branches (another account left / none left). In all of those the app now simply lands on the signed-out welcome screen, whose Sign-in button opens the modal. The remaining call sites are all direct responses to a click — the welcome screen's Sign in, the empty sidebar user chip, "Add account" in the profiles modal, and picking an account whose session has expired (that one is kept on purpose: the click would otherwise appear to do nothing) — `renderer/inventory.js`

- **The search field's ✕ now replaces the magnifier instead of sitting next to it** — main search bar and the rack view's "not stored" filter alike. One glyph, always in the same spot: the loupe stands down as soon as there is text, so nothing shifts as you type and the field keeps a single padding. The magnifier is hidden purely in CSS off the button's own `hidden` attribute, so there is no second state to keep in sync. The main button's `title=` tooltip was dropped in passing (project rule: `aria-label` only) and its label localised via `data-i18n-aria` — `renderer/inventory.html`, `renderer/css/70-detail-misc.css`, `renderer/css/30-racks.css`

- **Avatar colour on account creation — random HSL, generated once** (`renderer/inventory.js`). The onboarding avatar no longer picks from the 13-swatch palette: a single random colour is generated in HSL space (hue 0–360, S 65–85 %, L 42–58 % → vivid but legible) and converted to HEX (`generateRandomAvatarColor`/`hslToHex`). It is generated **once** when the profile step opens (never re-rolled on typing, initials change, re-render or validation), the **"Generate another colour"** button re-rolls it (only the colour changes — name/initials stay), and on finish it is saved as the account's custom HEX colour (`customColor` + `color_r/g/b` in Firestore → read back on future launches, and by Tiger Hub). Kept as the fallback when a photo is imported, so removing the photo later restores the initials avatar in that colour. The initials text colour is auto-contrasted by relative luminance (`readableTextOn`, threshold 0.58 → `#111111` / `#FFFFFF`). i18n: `setupAvatarShuffle` reworded to "Generate another colour".
- **Avatar initials — new shared contract** (`getInitials`, `renderer/inventory.js`). The background gradient stays derived from `acc.color`, never the name, so typing never shuffles the colour. Same rule to be mirrored in the mobile app and Tiger Hub.

## Fixed

- **The product card's ⓘ button didn't close the "Product info" card it had opened.** It called `openReorderPanel` unconditionally, so pressing it again just re-opened the same card. It now goes through `_toggleReorderPanel`, like the ⓘ on the spool detail and grouped-spools cards — the three behave identically.
