# Worklog — v1.10.19 (in progress)

## Added
- **Side-card slide-swap transition (spool detail + printer).** Switching from one open card to another now slides the new card in over a static, pixel-identical "ghost" clone of the old one (then removes the ghost), instead of an instant content swap. Generic `_ghostSwapPanel(panel, scrollSel)` uses 2 fixed z-index slots (real / ghost at 100, no growth); the ghost keeps its ids (so id-scoped styling like the hidden `#panelBody` scrollbar stays intact), is inert + removed in 400ms, mirrors the outgoing scroll position, and neutralises live media (camera `<iframe>`/`<webview>`/`<video>` → same-size dark placeholder, so no second stream spins up). Newly-opened spool cards start scrolled to the top. Wired on every user open: spool detail (table, grid, group panel, racks, RFID scan) and the printer side panel (`openPrinterDetail` when switching printers); programmatic same-card refreshes excluded. `renderer/inventory.js`, `css/70-detail-misc.css`

## Changed
- **Grouped view is the default; its toggle moved to Settings → Data.** The "group identical spools" switch is gone from the inventory toolbar — grouping is now the default view (already the case via `tigertag.inv.group !== "false"`). Users who prefer a flat list flip it off via a discreet toggle row ("Group identical spools") in Settings → Data. Same ids (`btnGroupInv`/`btnGroupInvLabel`) so the handler + Firestore sync are unchanged; removed the now-dead toolbar tooltip logic and `.group-toggle-btn` CSS. `renderer/inventory.html`, `renderer/inventory.js`, `css/70-detail-misc.css`
- **Inventory "Refresh" moved from the sidebar to Settings → Data.** It was a prominent primary button under the avatar, but the inventory already syncs live via `onSnapshot`, so a manual refresh is mostly a safety net. Now a discreet `ghost sm` button ("Resync inventory") in the Data card; same id (`btnSbReload`) so the handler + loading spinner are unchanged. `renderer/inventory.html`
- **"Choose a container" is now a side card, not a modal.** It tucks to the LEFT of the spool detail card (which keeps the right edge), integrated into `_syncPanels` as a back card (z-index 100, with its own `»` close tab). No more backdrop; Escape closes the picker first (then the spool card), and it auto-closes when the spool card closes or you switch spools. Container thumbnails enlarged to 64×64 to match the container image in the detail card (`.cc-body img`), with a `min-height: 80px` on each row so the bigger image fits without overflowing onto the group labels. Container weight now sits under the type (in the info column) instead of on the right. `renderer/inventory.html`, `renderer/inventory.js`, `css/70-detail-misc.css`
- **Settings panel polish.** Action buttons got a proper surface/border/radius look with clear hover (orange border + icon, subtle lift) and active/disabled states; the Auto-update toggle row and the collapsible Data/Tools headers now have a hover background. Replaced the leftover text glyphs (`⌥`/`⚑`) in the Tools buttons with SVG icons (debug→search, RFID→nfc, report→info; TD1S keeps its icon) and stripped `⌥` from the `debugOpenBtn` string (9 locales). `css/10-settings.css`, `renderer/inventory.html`
- **Debug panel now stacks to the LEFT of Settings** (like the printer config beside the printer panel) instead of opening over it. New `_layoutSettingsStack()` positions Settings as the right-edge anchor with Debug tucked to its left, each with its own `»` close tab; opening Debug from Settings keeps Settings open (no extra backdrop), closing Settings closes Debug too, and both stay glued on resize. `renderer/inventory.js`, `renderer/inventory.html`, `css/70-detail-misc.css`
- **"Spools not stored" rows now match the group panel's list-cards** — product image + tier badge on the left, name / material·brand / weight + colour bar on the right (reuses `_groupMemberCardInnerHTML`). Drag-to-rack is unchanged (`.rp-side-row` + `data-spool-id` kept). `renderer/inventory.js`, `css/30-racks.css`

## Fixed
- **"Spools not stored" chevron out of sync.** The chevron now tracks the panel at the same speed for every trigger (manual toggle, auto-open on entering Storage, and drag): the auto-open snaps closed then animates panel + chevron together, and during a drag the chevron slides to the edge via `right` (same `.28s` transition as the panel) and back on dragend — no more lag on reopen. `renderer/inventory.js`, `css/30-racks.css`
- **"Spools not stored" drag grabbed the image instead of the card.** The inner `<img>` had its native drag; clicking it dragged the image, not the spool. Children are now `pointer-events: none` + images `-webkit-user-drag: none` so the whole row is the single drag source, and the drag ghost is just the square, rounded thumbnail (`setDragImage` on `.thumb-wrap`, not the badge-stretched `.gp-member-thumb` div which made it a rectangle). `renderer/inventory.js`, `css/30-racks.css`
- **"Spools not stored" chevron handle reopened instead of closing.** The global click-outside handler closed the panel on mousedown, then the chevron's click toggle saw it as closed and reopened it. Whitelisted `#rpUnrackedTab` so the chevron owns its toggle. `renderer/inventory.js`
