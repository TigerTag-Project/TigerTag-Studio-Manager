# Worklog — v2.2.1 (in progress)

## Added
- **Public favorites + shared price/buy link between friends (`productShares`)** — friends now see **each other's favorites**: in a friend's view the **Favorites grid & table** show the friend's ★/❤ materials (read-only — their stock, price, clickable buy link), and each material's details show the friend's price + buy link (+ manual SKU/EAN). Importing (favoriting) a friend's material carries the price/link/SKU/EAN over. Because a Firestore read is all-or-nothing per doc, `/products` stays strictly private (the **note** is never exposed) and a new **friend-readable `users/{uid}/productShares/{keyHash}`** collection holds only the public slice: `favorite, liked, key, label, sku, ean, buyUrl, buyPriceHt`. The app mirrors it on every product write (`_syncProductShare` from `_writeProduct`/`_writeProductField`/delete; removed when the product is no longer a favorite nor carries price/link/code) + a one-time backfill (`_backfillProductShares`); in a friend view it subscribes to the friend's shares (`subscribeFriendShares` → `state.friendShares`), renders the friend's favorites (`renderProductsView`/`_renderFavesGrid`/`_renderFavesTable` read-only), fills price + buy link + SKU/EAN in the material details (`buildPanelHTML`), and copies them at import (`_carryFriendShare`). **Backend:** `productShares` rule (read owner/public/friend, write owner + field-whitelist incl. favorite/liked/label) in `firestore.rules` — **not yet deployed** — `renderer/inventory.js`, `TigerTag_Firebase_Backend/firestore.rules`, `docs/firestore-schema.md`, new i18n `pvFriendFavesEmpty` (9 locales)
- **New masterspool: PrintoMax 3D — Grey (195 g)** added to the container picker (`printomax3d_masterspool_grey`, brandId 18629) with its illustration — `data/container_spool/spools_filament.json`, `assets/img/spool_filament/printomax3d_masterspool_grey.png`
- **Bulk ★ Favorite / ❤ Love** — with spools multi-selected (grid or table), two icon-only toggle buttons (styled like the `.flag-toggle` in the Materials side card: muted outline when off, gold/red filled when on) add OR remove the flag across the whole selection at once. It's a real toggle keyed on the selection's aggregate state — filled only when EVERY selected product already carries the flag, and a click then removes it from all (else adds to all); deduped by group key, written via `_writeProduct` (`_bulkApplyFlag` / `_syncBulkFlagButtons`). Shown in the materials context (own inventory + friend view, where adding imports with provenance) — `renderer/inventory.js`, `renderer/inventory.html`, `renderer/css/70-detail-misc.css`, i18n `bulkFavorite` / `bulkLike` as button tooltips (9 locales)
- **Multi-select now works in a friend's inventory** — the checkboxes and the Select button are enabled in a friend view so you can bulk-favorite/like a friend's materials (each import stamped with provenance, like the single toggle). The bulk bar there shows **only** ★/❤ — Tags / Price / Delete are hidden (`is-friendview`) and `_bulkDeleteSelected` hard-guards against ever touching a friend's spools. Removed the friend-view early-returns in `_enterSelectMode` / `_toggleSelectAllVisible` and the button-hide/force-exit — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`

## Changed
- **Clicking your name in the sidebar acts like clicking your avatar** — the pseudo (`#sbName`) now runs the same identity action as the avatar (`_onSidebarIdentityClick`): toggle the account dropdown, or return to your own inventory when viewing a friend. Pointer cursor + hover cue on the name — `renderer/inventory.js`, `renderer/css/00-base.css`
- **Favorites views now available inside a friend's view** — the Favorites group (Grid + Table) stays visible while browsing a friend's inventory, so you can flip to your own favorites without leaving; only **"To order"** is hidden there (a reorder action on your owned stock). The bulk bar's `is-friendview` restriction (★/❤ only) now applies only to the friend's materials, not the viewer's own favorites table, which stays fully editable — `renderer/inventory.js`
- **View selector now has a sliding selection bubble** — within a segment (Inventory / Favorites / Printers) the active highlight slides smoothly between buttons instead of the background snapping on/off. Each segment has one absolutely-positioned `.view-toggle-ind` that `_positionViewIndicators` sizes + moves onto the active button (measured from live layout, so it copes with variable button widths); it re-fits instantly on first paint, language switch and window resize, and animates (transform/width) on view changes. Scoped to `.view-toggle-groups` so other reused `.view-toggle` controls are untouched — `renderer/inventory.js`, `renderer/inventory.html`, `renderer/css/70-detail-misc.css`
- **Buy/reorder actions now use the cart (caddie) icon** instead of the Shopify glyph — the buy-link buttons in the Product-info card, grouped-deck header, favorites grid/table, the "has a buy link" product badge, the toolbox reorder-buy row, the "To order" empty state, and the **"To order" view-selector** entry all switch `icon-shopify` → `icon-cart` (the same cart used in What's New). The genuine goodies-shop buttons (sidebar, friends panel) keep the Shopify icon — `renderer/inventory.js`, `renderer/inventory.html`
- **"Product info" button now toggles the card** — clicking the "Infos produit" button (grouped-deck header and the toolbox row) opens the Product-info side card, or **closes it** if it's already showing that same product. New `_toggleReorderPanel(r)` compares by `_spoolGroupKey` — `renderer/inventory.js`
- **Product-info "+ Material" button now matches the toolbox rows** — it's rendered as a real `.toolbox-row` (icon + label + trailing ⓘ) instead of a centered pill with a separate info box, so it looks and hovers like every button in the toolbox. `_wireReorderInfoTips` now also swallows pointer/click on the ⓘ (capture phase) so pressing it never fires the button — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`

## Fixed
- **"Detach" button showed where it shouldn't (incl. a friend's view)** — the cam-wall Detach button is toggled via the `hidden` attribute, but `.inv-add-btn { display: inline-flex }` (author style) overrode the UA `[hidden] { display:none }`, so `hidden` never actually hid it — it leaked into every non-cam view. Added `.inv-add-btn[hidden] { display:none }` so the attribute works again; `renderFriendBanner` also force-hides it in friend view as defence — `renderer/css/70-detail-misc.css`, `renderer/inventory.js`
- **In-stock spool count double-counted twin pairs** — a twin pair (two RFID chips for ONE physical spool) was counted as 2, so a product with 2 physical spools (one being a twin) showed "In stock: 3". A shared `_countPhysicalSpools` now collapses twin pairs (same pairing rule as `deduplicateTwins`), applied to the Product-info / deck stock badge (`_filamentStockCount`), the favorites & "To order" stock (`_stockCountByKey`), and the low-stock notification check (`_checkLowStockNotifs`) — `renderer/inventory.js`
