# Worklog — v1.10.27 (in progress)

## Added
- **More telemetry in `users/{uid}/telemetry/studio`.** New once-ever fields: `discordClickedAt` / `githubClickedAt` / `makerworldClickedAt` (first click on each sidebar community link — strict first-click-ever across devices: all three are gated by their synced `…Seen` flag, stamped once when the nudge is marked done), and `firstScaleAt` / `firstRfidReaderAt` (first time a TigerScale / RFID reader was connected — milestones like `firstSpoolAt`, complementing the existing current-state `scalesCount` / per-session `rfidReadersMax`). The account's first-ever Tiger Studio connection is already captured by the existing `firstSeen`. All new field names were added to the `telemetry` `hasOnly()` whitelist in the backend rules (deployed). `renderer/inventory.js` (+ `TigerTag_Firebase_Backend/firestore.rules`)
- **Community-link nudges (Discord / GitHub / 3D Files).** Each of the three sidebar community buttons carries a message-style red "1" badge + a notification inviting the user (join the Discord, star the GitHub repo, discover the free MakerWorld 3D files), until they act on it once. Click routing: while a nudge is pending, clicking its sidebar button opens the **notification center** (so they read the invite); once done, the button jumps straight to the link; clicking the notification itself opens the link AND marks it done. "Done" is a per-account, Firestore-synced flag (`users/{uid}.{discord,github,makerworld}Seen`) so the nudge never returns on any device — no verification that they actually joined/starred, the click is enough. Config-driven (one `COMMUNITY` map) so all three share the same code; the notification icons use the real brand look (purple/slate/teal rounded square, white logo). `renderer/inventory.js`, `renderer/inventory.html`, `css/10-settings.css`, `css/20-friends.css`, locales

## Changed
- **New vector `icon_speed.svg`** — a simplified car-speedometer (semicircle gauge + needle + hub), replacing the heavy raster PNG-in-SVG. True vector (378 B vs ~30 KB), so it masks crisply and themes (muted, etc.). `assets/svg/icons/icon_speed.svg`
- **Printer control card: Step / Speed dropdowns no longer stretch full-width** (Elegoo / Snapmaker / Creality). They filled the whole info column, while Bambu/Anycubic were already snug thanks to a fixed-width override in `anycubic.css` (`.acu-jog-extra .elg-ctrl-speed-select { width: 96px; flex: none }`). The base `.elg-ctrl-speed-select` now uses the same approach — a fixed `width: 110px; flex: none` — because `flex: 1 + max-width` does NOT reliably cap a native `<select>`. `css/57-elegoo.css`
- **Temperature icons (nozzle / bed / chamber) now use their dedicated SVGs** (`icon_nozzle.svg` / `icon_bed.svg` / `icon_chamber.svg`). The shared `SNAP_ICON_NOZZLE` / `SNAP_ICON_BED` / `SNAP_ICON_CHAMBER` (used by every brand's temp card) switched from inline line-art SVGs to masked `icon-nozzle` / `icon-bed` / `icon-chamber` (bed + chamber are true vectors, so they mask cleanly). Also fixed Elegoo's chamber pill, which was wrongly showing the **bed** icon. `renderer/printers/snapmaker/index.js`, `renderer/printers/elegoo/cards.js`, `css/70-detail-misc.css`
- **Creality control card now has a Speed dropdown** (like Bambu Lab / Snapmaker / Elegoo — it was the only brand missing one). It sets the feedrate override via `M220 S<pct>` (25–200 %, defaulting to the live `curFeedratePct`) over the existing Creality G-code path, shown with the `icon-speed` icon next to the Step dropdown. `renderer/printers/creality/index.js`, `renderer/inventory.js`
- **Refreshed the printer-card icons.** New, cleaner SVGs across the printer detail cards. In code: AMS humidity now uses a dedicated `icon-humidity` (was the generic droplets); the disable-motors button uses `icon-disable-motor` (was `icon-bolt`); and the jog "Step" / "Speed" text labels are now `icon-step` / `icon-speed` icons (with the text kept as a hover title) — across the jog control cards (Anycubic / Bambu Lab / Snapmaker / Elegoo / Creality). `renderer/printers/{anycubic,bambulab,snapmaker,elegoo,creality}/*.js`, `css/70-detail-misc.css`, `assets/svg/icons/`
- **Sidebar GitHub / 3D Files / Discord buttons: 40px tall, square when collapsed, bigger logos.** They're a fixed 40px in both states (matching the Notifications button + friend avatars) and collapse to a 40×40 **square** (was a slightly rectangular 44×34). Their brand logos are bigger and uniform — a shared 24px rule replaces the old per-button inline 16/18/20px sizes — so each reads like an app icon. `css/10-settings.css`, `css/00-base.css`, `renderer/inventory.html`
- **App notifications redesigned: icon + title + baseline, and the whole row is the action.** Local notifications (Discord nudge, "add an avatar") no longer truncate to one ellipsised line with a cramped CTA button beside it — they now read as a bold title + a wrapping muted baseline, and clicking anywhere on the notification triggers the action (open Discord / change avatar). The separate "Join" / "Add avatar" buttons are gone (the notification IS the call-to-action); update notifications keep their explicit "Restart" button. `renderer/inventory.js`, `css/20-friends.css`, locales
- **"Spools not stored" now lists empty (0 g) spools at the bottom.** The rolls that still have filament — the ones the user actually needs to rack — surface first; depleted spools sink below them. Stable sort in `getUnrackedSpools()` (so each group keeps its order, and the auto-fill pool, which drops empties, is unaffected). `renderer/inventory.js`

## Fixed
- **Printer side-card icons are now one uniform colour.** The jog-pad home icons were dark (`var(--text)`) while every other control icon (left home / disable-motors / LED / files, fan, filament-edit, Step/Speed, humidity) was `var(--muted)` — so the panel mixed black and light-grey icons. The jog-pad home buttons now use `var(--muted)` too (still turning white on hover), so all resting icons match; the deliberate accent colours (hover primary/white, fan-on green, LED amber, pause/stop) are unchanged. `css/57-elegoo.css`
- **No more text flash on load (stale hardcoded labels).** Many `data-i18n` elements shipped with a hardcoded English default that didn't match their actual `en.json` value (e.g. "View on GitHub" → "GitHub", "Mobile App" → "TigerTag Mobile Apps", "Join us on Discord" → "Discord"), so the first paint briefly showed the wrong/long text before `applyTranslations()` swapped it. The 37 mismatched HTML defaults are now aligned to the EN locale (the i18n fallback) — properly HTML-escaped, arrays/plurals skipped — so the very first frame already shows the definitive text (and for brand labels like GitHub/Discord, no swap at all in any language). `renderer/inventory.html`
- **Friend notifications no longer truncate.** Like the local notifications, the "X accepted your friend request" rows now wrap instead of cutting off with an ellipsis (`.notif-text` wraps globally). `css/20-friends.css`
