# Worklog — v2.1.1 (in progress)

## Added
- **Bulk-set a price from the Inventory view too** — the bulk bar's **Price** action now shows for a materials (spool) selection, not just the Products table. Applying writes the price to each selected spool's **product identity** (deduped via `_spoolGroupKey`, created through `_writeProduct` so the record + label exist). `_bulkEnterPriceMode`/`_bulkApplyPrice` are now context-aware (`is-materials` vs `is-products`; printers excluded) — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`
- **Sortable columns in the Favorites table** — click a header (Brand · Material · Name · Stock · Min. qty · To order · Price) to sort ascending/descending, with the same chevron indicator as the inventory / printer tables. Own persisted sort state (`state.favesSortCol`/`favesSortDir`, `tigertag.sort.faves`); wired via a delegated `th[data-fsort]` handler on `#invProductsView` since the table is re-rendered — `renderer/inventory.js`
- **9 new illustration SVG icons** for wider What's New (and general) use — `mail`, `tag`, `list-check`, `cart`, `coins`, `bug`, `filter`, `gift`, `shield-check` — each with its `.icon-<name>` mask class (24×24, consistent stroke style) — `assets/svg/icons/icon_*.svg`, `renderer/css/70-detail-misc.css`
- **Favorite provenance** — favoriting a friend's material (from the friend view) now records where it came from. `_toggleProductFlag` stamps `importedFrom {uid,name}` once at import time (never overwritten), persisted by `_writeProduct` — only the `uid` matters, `name` is a last-resort frozen fallback. The Product-info card shows a **prominent "Added from" identity block** under the header (avatar + small label over the pseudo in bold, styled like the sidebar account row, with a chevron affordance when clickable) whose **avatar + pseudo are resolved LIVE**: while the card is open, `_subscribeImportedProfile` opens a direct `userProfiles/{uid}` listener (world-readable to any signed-in user) so the name/photo stay current **even after the friendship ends** — `state.friends` is used first (it also carries the inventory key) and falls back to the live profile cache. The chip is **clickable → opens that friend's inventory** (`switchToFriendView`) when still a friend OR when their inventory is public; otherwise it stays inert (private inventory can't be read) but still shows live data, greying only if even the profile is unreadable (deleted account). `_refreshReorderProvenance` swaps just the chip on any profile/friends change (no panel rebuild) — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`, new i18n keys `productImportedFrom` + `productFromUnlinked` (9 locales)

## Changed
- **Product-info card layout** — the material **illustration is larger** (64 px), the **colour name moved to its own line** below the material (no longer truncated inline next to it), and the **❤ Liked / ★ Favorite toggles moved into the panel header**, left of the ✕ close button (out of the body identity row). The delegated flag handler now resolves the reorder row from the whole `#reorderPanel` (header included); header flags are painted from `_flagTogglesHTML` on render — `renderer/inventory.js`, `renderer/inventory.html`, `renderer/css/70-detail-misc.css`
- **Product-info "+ Material" button** — the "Create a TigerTag Cloud" button in the reorder card is relabelled **"+ Material"** (plus icon) with an ⓘ info affordance next to it: hovering shows a toolbox-style tooltip explaining it adds a new spool of the exact filament to the inventory without an RFID chip. Tooltips inside the reorder card are now wired (`_wireReorderInfoTips`, delegated on `#reorderPanelBody`, reusing the toolbox tooltip) — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`, i18n `productCreateCloud` relabelled + new `productCreateCloudTip` (9 locales)
- **What's New illustrations now use real SVG icon names.** The modal renders `it.icon` as a masked `.icon-<name>` and strips anything that isn't a class token — so the emoji icons that had been used silently fell back to `sparkle`. The v2.1.0 entries now use proper icons (`star-fill`, `cart`, `bell`, `coins`, `list-check`, `image`, `mail`), and every older entry that used an emoji (🏷️🖼️🛍️👀🎨⏰🗑️ → `tag`/`image`/`shopify`/`eye-on`/`palette`/`clock`/`trash`) was remapped too, so each item shows its own illustration instead of a generic sparkle. All 37 distinct icons now resolve to a real class — `data/whatsnew.json`
- **The low-stock reorder notification now jumps to the "To order" list** instead of opening the single product's card — `_openLowStockProduct` closes the notifications and `setViewMode("order")` — `renderer/inventory.js`

## Fixed
- **Renaming a rack now updates the Storage view immediately** (was: had to leave the view and come back). `_rackStructureSig` — which decides whether `_tryPatchRackSlots` can patch slots in place vs full-rebuild — didn't include the rack name, so a name-only change let the no-op slot patch skip the header rebuild while still advancing the render signature. The name is now part of the signature — `renderer/inventory.js`
- **The grouped deck no longer hides behind the printer side card.** `_syncPanels` positioned the group deck (and the container / reorder cards) relative to the spool-detail width only — so with a printer panel open but no spool detail, the deck opened at `right:0`, straight behind the printer panel. They now cascade off the **printer stack width** (`printerW + configW`) like the spool card already did, so the deck sits to the LEFT of the printer panel, fully visible — `renderer/inventory.js`
