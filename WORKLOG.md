# Worklog — v1.10.31 (in progress)

## Added
- Public-inventory showcase in friend view: when you open a friend whose inventory is **public** (`isPublic`), you now land straight on the Storage (rack) view rendered as a locked picture-mode gallery — no "not stored" panel, no colour/picture toggle (can't switch to the working colour view), just the racks as square material tiles. A regular (non-public) friend still opens on the Materials grid as before, and it never affects the owner's own views or saved preference. Friend `isPublic` is read from `userProfiles` in `loadFriendsList` (+ live via the per-friend listener) — `renderer/inventory.js` (`switchToFriendView` default view + `friendView.isPublic`, `_rackSlotView` force, `renderRackView` showcase gating: aside/toggle/unranked-stat hidden, `.rp-showcase` class)
- Rack slot-content view switch (segmented toggle in the storage header): the current **colour fill** (remaining-weight bar) vs a new **photo** view that is purely aesthetic — every slot becomes a bigger SQUARE (1:1) tile (64px tracks, roomy gallery grid like the unranked cards) — a full solid square in the filament colour (no weight bar → no at-a-glance usage info), with the material's illustration image layered on top (cover, uncropped since illustrations are square) when it has one; spools with or without an image look the same (full colour tile). Filled tiles get a solid frame + slight elevation (drop shadow) so even a WHITE material reads as an occupied tile vs an empty slot (which keeps its flat dashed outline) — same filled/empty affordance as the colour view; they don't react to hover. Switching mode forces a full rebuild so the masonry re-lays out for the new slot shape. Choice persists per device (`localStorage tigertag.rackSlotView`). Slot HTML is dispatched through `_slotInnerHTML` (used by both the full render and the surgical patch, so moves stay flicker-free in either view), and the mode is part of `_rackViewSig` so toggling re-renders — `renderer/inventory.js` (`_rackSlotView`, `slotPhotoInnerHTML`, `_slotInnerHTML`, header toggle + wiring), `renderer/css/30-racks.css` (`.rv-slotview-toggle`, `.rp-fill--full`, `.rp-photo`, no-hover rule), `assets/svg/icons/icon_image.svg` (new) + `.icon-image` mask, i18n `rackViewMode`/`rackViewColor`/`rackViewPhoto`
- Two rack drag-drop sounds (synthesized, reuse `_playTones`): a crisp ascending "snap into place" when a spool is dropped into a rack slot (moved/placed), and a gentle descending "un-dock" when a spool leaves a rack back to "not stored". Fired only on user drops (slot / unranked-strip / void), not on auto-fill/auto-store; skipped on no-op drops (same slot, or an already-unranked row) — `renderer/inventory.js` (`_soundRackMove`, `_soundRackRemove`, drop handlers)

Shared Shopify-bag icon — `renderer/inventory.html`, `renderer/inventory.js` (`sbShopBtn` via `_clickCommunityBtn`, `COMMUNITY.shop`, `btnGoodiesShop`, `shopSeen` load), `renderer/css/10-settings.css` (`.sb-shop-btn`) + `00-base.css` (collapsed) + `20-friends.css` (`.fp-shop*`, `.notif-ic--shop`), `assets/svg/icons/icon_shopify.svg` (new) + `.icon-shopify` mask in `70-detail-misc.css`, i18n `shopBtn`/`friendsShopTitle`/`friendsShopSub`/`notifShopTitle`/`notifShopText`. ## Fixed
- Multi-colour swatches (`.color-circle`) rendered broken: a bicolor/tricolor/rainbow filament showed its conic-gradient bleeding into a square that overlapped the round swatch, leaking the base colour at the corners (Chromium doesn't reliably clip a conic-gradient background to `border-radius`). Reworked the swatch: the colour/gradient now sits on an absolute child clipped by the round parent via `overflow:hidden` — a reliable circular clip for any gradient. Fixes the colour dot in the table, the detail panel circle, grid cards, group members — everywhere `.color-circle` is used — `renderer/inventory.js` (`colorCircleHTML`), `renderer/css/70-detail-misc.css` (`.color-circle` + `.color-circle-fill`)
- Identical-spool grouping was too loose for DIY (TigerTag / TigerCloud) chips: they often keep the default red in `color_r/g/b` while the real colour lives in `online_color_list` (e.g. a "white" spool) or the 2nd/3rd colour (rainbow), so a white, a red and a rainbow spool collapsed into one group. The group key now uses the FULL colour signature (`colorHex` + `colorHex2/3` + the online colour list) plus both aspects — only genuinely identical-looking spools group now — `renderer/inventory.js` (`_spoolGroupKey`)
- Table view: a collapsed spool group no longer reveals a member row inline when a search/filter matches it (you'd see the group's aggregate row — e.g. 1375 g / "2" — AND a member row like 833 g at once, which read as a duplicate and was out of step with the grid, which only ever shows the single group deck card). Group members now follow the group's manual expand state only; a filter still surfaces the group via its header, and expanding it still filters its members — `renderer/inventory.js` (table search-dim: member `show` = expanded && matches)
- Notifications panel `»` close tab no longer floats over a side card opened in its place. The tab kept the base `z-index: 103` (above the `.detail-panel` cards at 100/101) and lingers ~280ms while sliding out, so a card opening right after covering it showed the tab on top. Since Notifications always opens alone (closes every other side card first), its tab is now pinned below all cards (`#notifCloseTab { z-index: 99 }`) — `renderer/css/70-detail-misc.css`
- Storage / rack view no longer rebuilds the whole interface (flicker + scroll jumps back to top) on Firestore changes. Three layers: (1) `renderRackView()` guards with a signature of what it actually renders and bails on no-op echoes — adding a tag from the detail panel, which the rack view doesn't display, no longer touches it. (2) Moving a spool between slots (the common case) now patches only the affected pucks in place — no `innerHTML` rebuild, no masonry reflow, no image reload → zero flicker; per-rack header counts update in place, drag/click handlers on newly-filled cells are wired idempotently. (3) For changes that do need a full rebuild (rack added/resized/locked, spool added/removed, unranked set changed) the scroll position of both the racks column and the "not stored" side list is preserved — `renderer/inventory.js` (`renderRackView`, `_rackViewSig`, `_computeRackSnapshot`, `_tryPatchRackSlots`, idempotent `wireDragSources`/`wireRackSlotOpen`)
- Inline tag editing in the detail panel no longer flickers, scrolls the card to the top, or closes the autocomplete dropdown after each pick. Tags are part of the detail structural signature, so the Firestore echo of a tag write was rebuilding the whole card; the in-place chip render now re-syncs the stored signature so the echo takes the lightweight patch path — you can add several tags in a row with the dropdown staying open — `renderer/inventory.js` (`_renderTagChips`)
