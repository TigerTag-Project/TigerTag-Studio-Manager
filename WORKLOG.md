# Worklog — v2.0.1 (in progress)

## Added
- Filament **buy-links / reorder** feature — foundation + account country landed (in progress). (A) The product-identity key includes the **Type** (id_type Filament/Resin) so filament and resin never share an identity, and a new per-user **`products`** table (`users/{uid}/products/{keyHash}`, keyHash = cyrb53 hash of the identity key — one record per **product identity**, not per material) stores, **once per product** and shared by every identical spool (surviving spool deletion): a buy link, purchase price (stored HT), min-stock-in-spools, a free **note**, **tags**, **SKU + EAN** (auto-recovered read-only from the tag for TigerTag+, editable otherwise), and a display snapshot (`label` incl. the product `imgUrl`). `_writeProduct` does a **partial merge** so the reorder card, the tag editor and future favourite/valuation slices each update independently without clobbering the others. The collection is deliberately named broadly (`products`) — it will also back favourites/wishlist and stock valuation — the stored photo lets the future "to buy" list render a filament even when none is in stock — live-synced into `state.products` via `subscribeProducts`, with `_getProduct` / `_writeProduct` / `_filamentStockCount` helpers (owner-only Firestore rule in the backend repo `firestore.rules`, deployed + committed). The debug Firestore explorer gets a **"Products"** quick chip (`FSE_QUICK`). (B) The account modal gains a **country picker** (auto-saves `users/{uid}.vatCountry`, drives VAT rate + currency from the editable `data/vat-rates.json`, 30 entries, default FR) **and a HT/TTC price-entry preference** (`users/{uid}.priceInputMode`, auto-saved) — how the user types filament prices, asked once here instead of per link. Prices are **always stored tax-free (`buyPriceHt`)**: a product's pre-tax price is country-independent, so the TTC figure is derived at display time via `_vatPrices`, and changing country re-renders prices without ever rewriting the DB. Shared `_vatInfo` / `_vatPrices` helpers. (C) A **"Product info" side card** (per-product management: buy link, price, stock/min, note, tags) opens from a **"Product info"** toolbox row (info glyph) in the spool detail panel. A separate **"Reorder"** action (Shopify glyph, green) opens the product's **buy link (shop) directly** via `openExternal` — always shown, both as a toolbox row (`btnToolReorderBuy`) and a deck button (`.gp-reorder-btn--buy`); **greyed when no link is set** (`_hasBuyLink`), and in that state clicking it opens the product card instead so the user can add one. The buy-link presence is folded into `_panelCardSig` so the toolbox row's greyed state flips live. Internal identifiers of the product card stay `reorder`/`ro-`/`#reorderPanel`.) It also opens from a **"Product info" button at the top of the group deck** (above the first member — the whole deck shares one product identity, so it targets `g.rep`). It's a distinct 3rd card that docks to the LEFT of the material detail — or to the LEFT of the group deck when that's open, never on top of it (`_syncPanels` offsets it by the group width; z 99 tucks it behind the deck): shows the product identity (real spool photo, colour-swatch fallback) + current stock (`_filamentStockCount`), edits the buy link via a **Shopify button** (the URL is never shown as text — grey when unset → click reveals an inline editor to add one; green when set → click opens the shop via `openExternal`; a pencil re-opens the editor to change it), the price in the account's HT/TTC mode — shown as read-only text ("16,20 €", locale-formatted via `_fmtMoney`) with an edit pencil that reveals the number input + a ✓ to save (never an always-open field; mirrors the buy-link pattern), a chip shows the mode and the other side + VAT rate derive live via `_vatPrices`; stored as HT — and a **Stock** section: a full-width current-stock pill ("In stock: N / min", amber when below target — `_reorderUpdateStock`) above a "Minimum stock" row whose value shows as text with an edit pencil → inline input + ✓ (same toggle as the price/link; `_reorderUpdateMinDisplay`), plus **SKU + EAN** (auto read-only from the tag for TigerTag+, editable otherwise). **No Save/Remove buttons** — everything auto-saves on the fly (price/min/link on their ✓, note/SKU/EAN on blur, tags on each change). The card is per PRODUCT identity, so navigating between spools of the SAME product (e.g. clicking members of one group deck) keeps it open; it only closes for a different product. `refreshOpenDetailReorder` patches the card's read-only bits live on a products/country change without clobbering an in-progress edit. A product with a saved buy link gets a **Shopify badge** on its illustration (grid card, grid group deck + table thumbnail — `_hasBuyLink`/`_buyLinkBadgeHTML`, folded into `_rowSignature`/`_groupGridSig`; hidden in friend view). (D) **Below-min-stock alerts** ✅ — a product whose live spool count drops below its `minStockSpools` shows an amber "In stock: X / min" pill at the top of its group deck (above the action buttons) AND raises a **local notification** (`_checkLowStockNotifs` → one auto-managed `lowstock:<hash>` notice per product: **product illustration** (falls back to an amber box icon), product name, "X/min left", chimes once, clears on restock; clicking opens the product card). Re-evaluated on every inventory/products change. Backend `firestore.rules` deployed + committed. Remaining: an "À racheter" shopping-list view — tracked in memory `project-filament-buy-links`. `data/vat-rates.json`, `renderer/inventory.js` (`_spoolGroupKey`, products + VAT + tag helpers, `subscribeProducts`, `saveAccountVatCountry`, account-modal wiring, `openReorderPanel`/`_renderReorderPanel`/`_saveReorder`/`refreshOpenDetailReorder`, `_syncPanels`), `renderer/inventory.html` (country select + `#reorderPanel`), `renderer/css/60-modals.css` (`.eac-field-hint`), `renderer/css/70-detail-misc.css` (`.ro-*`), `TigerTag_Firebase_Backend/firestore.rules`
- **Bulk tag editing** from the multi-select bar (Shopify-style) — with one or more spools (or printers) selected, a new **Tags** button opens the existing tags modal for the whole selection. It reuses the per-item modal: `getTags()` returns the tags common to ALL selected items (intersection), and on save the diff is applied to every item — newly checked tags are added to all, common tags unchecked are removed from all, tags on only some items are left untouched (search/create a new tag adds it to all). Generic over spools (`_writeSpoolTags`) and printers (`_writePrinterTags`). New `_openBulkTags` + `ctx.title` support in `openTagsModal` — `renderer/inventory.html` (bar button), `renderer/inventory.js`
- Header **master checkbox** in both tables (materials + printers) — a Shopify-style ⬚ in the selection-column header that selects/deselects **all currently-visible (filtered) rows** in one click (replacing the removed "Select all" button, and honouring the active search/filters). Shows a checkmark when all visible are selected and a dash when only some are; stays in sync as the selection or filter changes. `_toggleSelectAllVisible` + `_syncSelAllHeader` (called from `_updateBulkBar`, the printer render, and the materials filter pass) — `renderer/inventory.html`, `renderer/inventory.js`, `renderer/css/70-detail-misc.css`
- Bambu Lab print thumbnails — the current print's model preview now shows in the printer table's Preview column AND in the Bambu side card (like the other brands, which previously left Bambu blank). Fetched from the printer over FTPS (implicit :990, `bblp` / Access Code, self-signed cert) per `bambulab/PROTOCOL.md §11`: find the current `.3mf` by exact name — the actively-printing slice lives in `/model/` (then `/cache/`, root, `/data`, `/data/Metadata` as fallbacks), matched against `gcode_file`/`subtask_name` across firmware-variant filenames (`{stem}.gcode.3mf` / `.3mf` + spaces→underscores, mirroring the battle-tested bambuddy resolver) — download it (it's a ZIP), extract `Metadata/plate_{N}.png`. The fuzzy fallback normalizes spaces↔underscores and never picks an arbitrary `.3mf` (a wrong-model thumbnail is worse than none). Handles the Bambu PASV quirk (it answers host `0.0.0.0` → rewritten to the control host) and picks the plate from `plate_idx`. Fetched once per file while a job is active OR finished (throttled, retried every ~10s) — a finished plate keeps showing "what just printed" until the printer goes back to idle (or the job failed/errored); cloud-mode printers (no LAN IP) are skipped. Validated end-to-end against a real A1. New deps `basic-ftp` + `yauzl` (both pure-JS, no native rebuild) — `main.js` (`bambulab:fetch-thumbnail` IPC: FTPS + PASV rewrite + fuzzy `.3mf` match + yauzl extract), `preload.js` (`bambulab.fetchThumbnail`), `renderer/printers/bambulab/index.js` (`conn.data.printPreviewUrl`, `_bblFetchThumbnail`, trigger in `_bblMerge`), `renderer/printers/bambulab/cards.js` (side card shows the preview when active), `renderer/inventory.js` (`_getPrinterJob` bambu `thumbUrl`)

## Changed
- Editing a spool's TD (the TD chip in the detail panel + the toolbox "Scan TD") now opens a dedicated **"Update TD"** modal that changes *only* the TD value — never the colour. It reuses the existing, still-wired TD-only modal (`#tdEditModalOverlay`), which `openTdEditModal` had been short-circuited to redirect into the colour+TD modal; restored its real open path (prefill + TD1S sensor state), dropped its colour-writing "All" button (kept a single full-width "Update"), and made the scanned HEX display read-only. Added a hold-to-confirm **"Clear TD value"** footer (shown only when the spool has a TD) so the TD can be removed from the modal — an empty field is rejected by design; it reuses the toolbox's clear logic (`_clearSpoolTd`, now shared via the edit-modals ctx). The colour+TD modal (`openColorEditModal`) is unchanged and still used for filament creation + the toolbox "Scan colour" — `renderer/IoT/td1s/edit-modals.js`, `renderer/inventory.html`, `renderer/inventory.js`, `renderer/IoT/td1s/td1s.css`
- Native-app feel: the whole UI is now clickable but not text/element-selectable — clicking or dragging anywhere no longer leaves a blue selection highlight. Applied globally via `user-select: none` on `body`, with exceptions so content users actually read/copy stays selectable: form fields (`input`/`textarea`/`select`/`[contenteditable]`), code/JSON blocks (`pre`/`code`), and anything opted in with a `.selectable` class — `renderer/css/00-base.css`
- Print preview now persists when a job is **finished** (not just while active) across every brand — the printer table's Preview column and the side card keep showing "what just printed" until the printer goes back to idle. Shared `_DONE_STATES` set (finished/complete/completed) drives it in `_getPrinterJob`; per-brand: Bambu/Elegoo/Snapmaker/FlashForge already retain their preview field (only the active-only gate was lifted, `bambulab/index.js` also keeps fetching the `.3mf` once finished), Anycubic now keeps its last `printThumb` while finished instead of nulling it, and Creality's table thumbnail switched from the (post-print stale) live camera snapshot to the file's own pre-sliced thumbnail (new exported `creCurrentThumbUrl`, camera kept only as an active fallback) — `renderer/inventory.js`, `renderer/printers/{bambulab,anycubic,creality}/index.js`, `renderer/printers/{bambulab,elegoo,snapmaker,flashforge}/cards.js`
- README: title is now "Tiger Studio Manager 2" (version-2 milestone); refreshed the outdated "Project structure" tree (services/nfc-process.js, IoT/, rfid_protocol/, anycubic + shared printer files, 10 CSS files, data/ subfolders, scripts/, docs/, CODEMAP-main.md, ROADMAP.md, updated CI comment) and fixed "5 → 6 supported brands"; also documented the printer `tags` field in `docs/firestore-schema.md` (+ mirrored a cross-app note into the backend repo README) — `README.md`, `docs/firestore-schema.md`
- Documented an automatic SemVer bump policy in `CLAUDE.md` (release ritual): decide MINOR (new feature/capability in Added) vs PATCH (only fixes/tweaks/i18n/internal) from the WORKLOG contents, MAJOR stays human-confirmed; the post-release bump is now framed as a PATCH placeholder corrected at the next release if warranted — `CLAUDE.md`

## Fixed
- Storage view's **"Auto-organize"** label always showed in English regardless of the app language — its span used `data-i18n` with a hard-coded English default, but the rack view is rendered dynamically and `applyTranslations()` doesn't re-run over it (every other rack label already uses `t()` at render time). Now renders `${t("autoOrganizeTitle")}` (the translations existed in all 9 locales; they just weren't applied) — `renderer/inventory.js`
- The rack view's ⓘ info tips (**Auto-organize**, the "no space" hint) rendered their bubble as a CSS `::after` positioned inside the card — it got clipped by the panel's overflow and slipped behind the interface. They now reuse the toolbox's shared `#toolInfoPop` (body-appended, `position: fixed`) via `_wireRackInfoTips` → `_showToolInfoTip`, so the bubble escapes the overflow/stacking and floats above everything — `renderer/inventory.js`, `renderer/css/30-racks.css`
- FlashForge open-frame printers (e.g. **AD5X**) showed a bogus "Door closed" badge — they have no door, yet the firmware permanently reports `doorStatus: "close"`. The door chip is now gated on the model catalog's `Enclosed` feature (`data/printers/ffg_printer_models.json`), so it only shows for enclosed models (5M/5M Pro/A5/Creator 5/5 Pro) — `renderer/printers/flashforge/cards.js`
- Printer table "Ends at" column was always blank for **Snapmaker** and **FlashForge** prints — `_getPrinterJob` had no `remainSec` branch for them. Snapmaker/Moonraker exposes no direct remaining time, so it's now derived from the slicer's `estimated_time` minus elapsed `print_duration` (falling back to a progress-based extrapolation); FlashForge's `printEstimated` already carries remaining seconds, so it's used directly — `renderer/inventory.js`
- Printer table Preview column: the thumbnail looked cramped against the row edges on a **finished** job vs a printing one. A finished job has no progress bar / % / filename, so its row is shorter — and the preview cell's near-zero vertical padding (`1px`) let the fixed 60px thumbnail nearly fill the short row, while it sat comfortably inset on the taller printing rows. Bumped the cell's vertical padding (`.pt-td--jobthumb`) so the preview keeps consistent breathing room regardless of row height — `renderer/css/40-printers.css`
- The multi-select bar's **Delete** button (materials + printers) showed no hold-to-confirm fill animation, so it looked broken (users clicked and, seeing nothing, gave up before the 1.5 s hold completed). The button is already solid red, but the default `.hold-progress` fill is also red (`var(--danger)` @ .35 opacity) → invisible red-on-red. Added a contrasting dark sweep for it (`.bulk-btn--danger .hold-progress`) — `renderer/css/70-detail-misc.css`. Also codified the rule in `CLAUDE.md` (Hold-to-confirm buttons): the fill is mandatory and must visibly contrast the button's own background
- Launching straight into a printer view (saved as the last view) showed the **materials** filters (Material / Version) to the right of the search bar instead of the printer ones (Brand / State / Tags), with the materials search placeholder. The boot view-restore block replicated `setViewMode`'s button state but never applied the `printer-filters` body class / repopulated the printer filters / synced the placeholder — now it does — `renderer/inventory.js`
- Release naming: the new `prepare-release` job created the GitHub Release with its name defaulting to the tag (`v2.0.0`) instead of the historical `2.0.0`. Both `softprops/action-gh-release` steps now set `name:` to the version with the leading `v` stripped (`${GITHUB_REF_NAME#v}`, `shell: bash` on the Windows runner) — `.github/workflows/build.yml`. (The already-published v2.0.0 release was renamed to `2.0.0` via the API.)
