# Worklog — v2.3.1 (in progress)

## Added
- **Click-to-copy SKU & EAN in the Product-info card (no button chrome)** — replaced the big copy button on each reference row with the same épuré pattern as the To-order list. A read-only auto TigerTag+ ref is itself the click target (faint copy glyph on hover + "Copied!" flash). An **editable** SKU/EAN shows a click-to-copy value plus a **single toggle button** that flips a grey edit pencil ⇄ a green ✓ (no separate always-visible input, no separate validate button): click the pencil to edit, commit with Enter or by clicking the green ✓; clicking an empty value jumps straight into edit. Shared `_wireRef` (copy + single-button edit toggle) + `_reorderUpdateRefDisplay` + `_flashCopied`; `[hidden]` guards so the input only shows while editing; tooltips via `copyLabel` / new `reorderEditRef`. Dropped the now-dead `makeCopyBtn` / `SVG_CHECK` helpers — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`
- **"To order" info button matches the cart button size** — the ⓘ info button had no dedicated style so it fell back to the global 40px-tall `<button>` default, making it bigger than the 34×34 cart/× buttons. It's now an explicit 34×34 light action button, aligned with the rest of the line actions — `renderer/css/70-detail-misc.css`
- **"To order" info button toggles the Product-info card** — reclicking a line's ⓘ button while its card is already open (same product) now closes it, instead of just reopening it (`_toggleReorderPanel` for the order-tab ⓘ; favorites clicks still always open) — `renderer/inventory.js`
- **Click-to-copy SKU in the "To order" list (no button chrome)** — the SKU value itself is the click target: no persistent copy button cluttering every line, just a faint copy glyph that fades in on hover and a "Copied!" pill that floats up over the SKU on click. Delegated `[data-copysku]` handler → clipboard + `_flashCopied` (reusable floating-flash helper). New i18n `copiedFlash` (9 locales) — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`
- **Aspect filter in the search toolbar** — a new "All aspects" selector (between Material and Version) filters the inventory by finish/aspect, matching **either Aspect 1** (finish: Matte, Silk, Carbon…) **or Aspect 2** (colour arrangement: Bicolor, Tricolor, Rainbow…). The dropdown lists the union of both aspect columns actually present (empty "-"/"None" dropped), works across the grid/table/rack views and the Favorites views (scoped to the favorites there, from `label.aspect` + new `label.aspect2`), and is hidden in the printer views (materials-only, like Version). New i18n `filterAllAspects` (9 locales) — `renderer/inventory.js`, `renderer/inventory.html`, `renderer/css/40-printers.css`
- **Product "business card" side card** — clicking a favorite that has **no live spool** (e.g. a friend's favorite they don't have in stock) opens a card that mirrors the **Materials side card** using identical markup: illustration, identity, **★/❤**, **Colours & Aspect** (56 px circle + aspect chips), **Weight** (full 1 kg / 100 % bar), **Print parameters** (nozzle / bed / drying / TD / density) and **Details** (Product ID, Type, Brand, Series, Name, Material, Diameter, SKU, Barcode) — omitting spool-only fields (Tag type, UID, Updated, Manufactured, location) and all RFID/weight-edit functions. Price (in the user's tax mode) + clickable buy link + provenance + a "not in this inventory right now" note round it out. The **★/❤ are tied to MY account** (not the friend's): they show/toggle whether *I* favorited it and clicking imports it (writes directly to my `products/{keyHash}` + carries price/link/SKU-EAN), via `_toggleProductCardFlag`. The card renders the FULL material info from the friend's product's **`cloudSeed`** (read directly — the card does `normalizeRow(cloudSeed)` → real colours, temps, id_material, diameter, sku/ean, product id); print params come from the chip's own temps (else the material DB's recommended). Also shows the material's **video** (YouTube thumbnail → browser, or inline MP4) and **document links** (MSDS / TDS / RoHS / REACH / food PDFs) from the seed's LinkXXX fields, like the materials card. New `#productCardPanel` (`openProductCard`/`_renderProductCard`), docked in `_syncPanels` to the RIGHT of the reorder ("Infos produit") card when both are open (the reorder card tucks left) and layered ABOVE it (z-index 100 > 99) so it reads as the front card, **no title bar / ✕** (closed via the » tab, like the Materials card); wired from the friend favorites grid/table (in-stock still opens the grouped deck) — `renderer/inventory.js`, `renderer/inventory.html`, `renderer/css/70-detail-misc.css`, new i18n `pcNoStock` (9 locales)

## Changed
- **Simpler stock line in the "To order" list** — the sub-title under each product name went from "5 required · 2 in stock" to a compact "Stock: 2 / 5" (in-stock / required). New i18n `pvStockRatio`; retired `pvRequired` / `pvInStock` and their `.pv-qty-req` / `.pv-qty-sep` styling — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`
- **No separator line under the Product-info title** — dropped the `.panel-header` bottom border for the reorder card (`.ro-panel`); the identity header already frames the content — `renderer/css/70-detail-misc.css`
- **Cleaner Product-info (reorder) card header** — the clickable identity block was cramped: a long colour name ("Dual Matte Sunset") collided with the open-card chevron and the lines had no clear rhythm. The text column now takes all width left of the chevron so every line (brand / material / colour) truncates cleanly; the colour name sits on its own line prefixed by a proper colour swatch (handles bi/tri/rainbow via `colorCircleHTML`); the "open product card" chevron became a subtle round button, vertically centred, that lifts on hover; thumbnail trimmed to 58 px and vertical spacing evened out — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`
- **Toolbar filter dropdowns are now app-styled (custom, not the OS-native menu)** — the Brand / Material / Aspect / Version / Tag selectors AND the grid-view **Sort** selector (`#gridSort`) keep their `<select>` (value + all existing populate/change logic) but it's visually hidden and driven by a styled button + popup list (`_enhanceSelect`): rounded card, hover rows, checked/active option, matching the app instead of the grey macOS dropdown. The custom button label resyncs on a programmatic sort change (`_syncGridSort`) and on a language switch (`applyTranslations` refreshes every `.csel`). The `#typeFilter` show/hide rules now target the custom wrap via `:has()` — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`, `renderer/css/40-printers.css`
- **Favorites-view filters list only what's in the favorites** — in the Products (Favorites / To order) views the Brand / Material / Tag selectors are now populated from the favorites products (own products or the friend's, limited to liked/favorite) instead of the whole inventory. `populateQuickFilters`/`populateTagFilter` take a `_favesForFilters()` source in products mode; `renderProductsView` refreshes them so they stay scoped (incl. on friend-products load) — `renderer/inventory.js`
- **Bulk ★/❤ buttons match the other bulk buttons** — the Favorite / Love toggles in the bulk bar are now `.bulk-btn` (icon + text, same height as Tags / Price / Delete) instead of icon-only squares; still toggles (coloured icon that fills + a tinted button when the whole selection carries the flag) — `renderer/inventory.html`, `renderer/css/70-detail-misc.css`
- **Toggle the Product card from the Product-info (reorder) panel** — its identity header (illustration + brand/material/colour) is a clickable button (chevron + hover) that opens the read-only Product "business card" for the same product, and closes it when reclicked while already showing that product (`_openProductCardFromRow` toggles on `_productCardData.id`, reusing the live product record or a synthesised one). New i18n `reorderOpenProductCard` (9 locales) — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`
- **A friend's favorite opens the Product card first** — clicking any favorite in a friend's Favorites grid/table now opens the product "business card" (whether or not the friend has it in stock), instead of the grouped deck for in-stock ones — `renderer/inventory.js`
- **Friends' favorites are read DIRECTLY from their `products`** — no duplicated projection. `products` is now friend-readable (owner/public/friend, same policy as inventory & racks), so viewing a friend subscribes to their real products (`subscribeFriendProducts` → `state.friendProducts`) and everything — the friend's favorites grid/table, the material's price/buy link, the product "business card", and import — reads their live product doc, always ISO with them. Removed all mirroring (`_syncProductShare`, `_backfillProductShares`) and the shares subscription. **Tradeoff (user-chosen):** the product `note` lives in that doc and is therefore technically readable by a friend (the app never displays it); making it strictly private would need a separate owner-only doc. **Backend:** `products` read opened to owner/public/friend (deployed); `productShares` deprecated — `renderer/inventory.js`, `TigerTag_Firebase_Backend/firestore.rules`, `docs/firestore-schema.md`

## Fixed
- **Switching favorite closes a stale Product card** — in the Favorites view, opening a favorite's Product-info (reorder) card while a Product "business card" was open for a *previous* product left that card lingering on the old product (it doesn't track the reorder card). `openReorderPanel` now closes the Product card when it's open for a different product identity (`_productCardData.id !== _productKeyHash(r)`) — `renderer/inventory.js`
- **Favorites Material filter now lists plain materials (not "PLA Basic")** — in the Favorites/To-order views the Material selector was populated from `label.material`, which carries the aspect suffix (`materialWithAspect` → "PLA Basic", "PLA Silk"…), so it listed aspect-specific duplicates instead of clean materials like the inventory view. It now uses the raw material name (`_faveMaterialRaw`: new `label.materialRaw` on newer products, else resolved from the product's `cloudSeed.id_material`, else the label). The matcher already accepted a raw value (exact or `"<mat> "` prefix), so selecting "PLA" matches every aspect — `renderer/inventory.js`
- **Product card provenance now uses the prominent "Added from …" block** — the read-only Product card showed its origin as a tiny grey one-liner (`.pc-from`) while the reorder ("Infos produit") card used the full avatar + label + name block (`.ro-from`). The Product card now renders the exact same `_importedFromHTML` block (wrapped in `.pc-from-wrap` for the section padding) and it's clickable → jumps to that friend's inventory when they're a friend — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`
- **Product-info header hover glitch** — the clickable identity header used arbitrary negative margins (`-6px`) on a `width:100%` button, so its hover box overflowed the panel and looked misaligned. Switched to a clean full-bleed hover (negative margins = the body's 16px padding, re-inset by padding), `box-sizing:border-box`, no border — `renderer/css/70-detail-misc.css`
- **Product card shows the PRODUCT's own provenance** — the "Added from …" line now reads the product's `importedFrom` field (where its owner grabbed it), not the friend you're currently viewing. So on a friend's product card, you see who THE FRIEND imported it from (a third person); on your own, who you imported from — `renderer/inventory.js`
- **Importing a friend's favorite no longer creates a duplicate / loses provenance** — the product card's ★/❤ used a bespoke write that stored the product WITHOUT its `cloudSeed`, so the imported product was incomplete: managing it later from your own favorites re-derived a DIFFERENT keyHash (from the lossy label) and a second product was created on the next write, with no `importedFrom`. `_toggleProductCardFlag` now delegates to the SAME `_toggleProductFlag` used everywhere else — it builds the full row from the friend's `cloudSeed`, so the keyHash matches exactly what favoriting from your own inventory computes (no duplicate, merges with a real spool later), stores the cloudSeed, keeps the interest coupling, stamps provenance, and carries the shared price/link/SKU-EAN — `renderer/inventory.js`
- **Own favorite with no spool now always opens the editable Product-info card** — clicking a favorite you no longer have in stock opens the reorder (Product info) side card so you can still set price / min stock / note / buy link / tags. It synthesises the row from the product's cloudSeed (else its label) via `_productAsRow`, so the card opens even when no live spool exists (was: nothing happened if the product had no cloudSeed) — `renderer/inventory.js`
- **Favorites grid & table now show prices in the user's tax mode** — they always displayed the stored HT price (labelled "HT") even when the account is set to TTC. A shared `_favePriceHTML` now derives the TTC figure from the country VAT rate when the user picked TTC (like the reorder card / "To order" list already did), tagging it HT/TTC accordingly — `renderer/inventory.js`
