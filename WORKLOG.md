# Worklog — v2.4.1 (in progress)

## Added
- Inventory **Table view**: a new **sortable Price** column (after Capacity) showing each spool's product price (`buyPriceHt`) in the account's currency + HT/TTC mode ("-" when unpriced). Single rows and group-header rows both carry it; the row/group signatures include the price + display mode so cells refresh on change; sorting handled in both `sortRows` and `_sortGroupedItems` (reads the price off the product) + a "Price" option in the grid sort select — `renderer/inventory.html` (thead + sort select), `renderer/inventory.js` (`_priceCell`, `_tableRowInnerHTML`, `_groupHeaderInnerHTML`, signatures, sort)
- New **"Stock Value"** header stat card (after "Stock" kg): the total worth of the current stock in the account's currency — each spool valued at its product's `buyPriceHt`, prorated by remaining-weight fraction (`weightAvailable / capacity`), shown in the account's HT/TTC mode with its VAT symbol — `renderer/inventory.js` (`renderStats`), i18n `statValue` / `statValueMini`
- "Buy me a coffee" support (https://buymeacoffee.com/benoitl) surfaced at four entry points, all sharing one `COFFEE_URL` constant and reusing the existing community-nudge system. Uses the **official Buy Me a Coffee brand assets** (never restyled): in the sidebar, a community-style button matching the others (full-width yellow pill, official cup logo `assets/svg/logos/logo_buy_me_coffee.svg` — watermark-stripped + moved out of `assets/img/spool_filament/` — as its icon, localized typed label `coffeeBtn`; collapsed → 40px yellow square with just the cup); the full button wordmark (`assets/svg/logos/logo_buymeacoffee.png`) in Settings → About and the What's New footer; a mono coffee-cup glyph (new `assets/svg/icons/icon_coffee.svg`) in the notification centre. The fourth entry point is a **delayed** notification/badge that only appears after 3 days of use (`delayDays` + first-use timestamp in localStorage `tigertag.firstUse`) — asking a warm, engaged user rather than begging on day one. Any entry point opens the page and marks the nudge done (per-account `coffeeSeen`, Firestore-synced) so it never pesters someone who already visited — `renderer/inventory.js`, `renderer/inventory.html`, `renderer/css/10-settings.css`, `renderer/css/60-modals.css`, `renderer/css/70-detail-misc.css`
- Backend (separate repo): `coffeeClickedAt` added to the `telemetry` field whitelist so the coffee-click analytics write isn't rejected — `TigerTag_Firebase_Backend/firestore.rules` (needs `firebase deploy --only firestore:rules`)
- Reorder ("to order") view is now a two-zone, Amazon-style cart: the **active cart** (products short of their min-stock) and a **"saved for later" shelf**. A non-destructive `savedForLater` flag on the product doc (private to `users/{uid}/products` — no Firestore rule change; never mirrored to `productShares`) moves a line between zones, either via an Amazon-style **text action** under the item ("Save for later" / "Move to cart" — no ambiguous icon next to the buy-cart) or by **drag-and-drop**: a grip handle (new `assets/svg/icons/icon_grip.svg`) drags a line to **reorder within a zone** or **across to the other zone**, persisting a per-product `sortRank` (native HTML5 DnD, same pattern as the printer-grid reorder; both zones stay droppable even when empty). The payment total counts the active cart only — `renderer/inventory.js` (`renderProductsView` order branch, `_orderLineHTML`, `_renderOrderTab`, `_applyOrderDrop`/`_persistOrder` + DnD wiring on `#invProductsView`, `_writeProductField`), `renderer/css/70-detail-misc.css`

## Changed
- Loving a product (❤) now **forces a minimum stock of ≥1** so it's automatically tracked for reorder — it lands in the "to order" cart as soon as it's out of stock. Coupled in `_coupleFlags` (never lowers an existing higher min; the forced min is carried into the persisted patch so it survives the Firestore echo) — `renderer/inventory.js`
- The four header status hovers (TD1S, TigerScale, backend cloud, TigerPod) now read as one family: same bubble chrome/position/font, each with a **state-coloured dot** (green connected · red disconnected · muted idle) + the full text. The TigerPod popup was realigned from a centred, differently-styled box to the shared right-anchored bubble (no more name truncation); the `.health` tooltips gained the matching dot via a `--tip-dot` radial-gradient — `renderer/css/60-modals.css`
- Header device indicators rebranded to product names: the **NFC-reader pod** (`#rfidReadersBar`) disconnected hover now reads **"TigerPOD not connected"** (was "no reader"; `rfidNoReader`) and its per-reader rows say "NFC #1/#2" (was "RFID #"); the **scale** icon reads **"TigerScale not connected"** (`scaleHealthNone`, + static `data-tooltip` fallback). The TD1S icon keeps its own "TD1S" wording (it's a separate device). Uses the app's custom `data-tooltip`/pod-popup, never a native `title` — i18n value changes only + `renderer/inventory.html`, `renderer/inventory.js`
- Add-product panel reorganised: **Type, Diameter, Weight and Unit are now always visible** (below the Nozzle/Drying cards) instead of buried in the Advanced section. The **duplicate Type select** is gone — the advanced mirror `adpTypeAdv` (and its basic↔advanced sync) was removed, leaving a single canonical `adpType`. The Advanced toggle now reveals only the deeper fields (Aspect 2, nozzle/bed/dry temps, TD, RFID/NFC preview) — `renderer/inventory.html`, `renderer/inventory.js` (`openAddProductPanel`, `_adpApplyMaterialDefaults`, `_ADP_CSEL_IDS`, listeners), `renderer/css/60-modals.css`
- Terminology: every **user-facing** "RFID" is now "NFC" (the tech is NFC/NTAG) — all 9 locale files (value strings only) + the few hardcoded UI labels in `renderer/inventory.html` (NFC Tag Tester, "Open NFC tester", NFC Data, Dual NFC Reader, …). Internal identifiers are intentionally untouched (i18n keys like `burnRfidBtn`, variables, IPC channels `rfid-*`, Firestore `rfidList`, `icon_rfid.svg`, code comments). The **`OpenRFID`** firmware setting name (Snapmaker/Paxx tutorial) is preserved verbatim — it's the literal label the user must find in their firmware
- Add-product panel: the plain identity `<select>`s (Type, Aspect 1/2, Diameter, Unit) now use the app-styled dropdown (`_enhanceSelect`) instead of the native OS option list — the button keeps the exact `.adp-select` field chrome, only the open menu becomes a dark on-theme popup, consistent with the toolbar filters — `renderer/inventory.js` (`_ADP_CSEL_IDS`/`_adpRefreshCsels`, enhanced at init + resynced on open/material-change), `renderer/css/60-modals.css`
- Windows code-signing CI now targets the rebuilt Azure Trusted Signing account — `.github/workflows/build.yml` (`trusted-signing-account-name` `TigerTagSigning`→`TigerTagStudioSigning`, `endpoint` `weu`→`neu.codesigning.azure.net`; the account was recreated in North Europe after the free-trial subscription lapsed and West Europe stopped onboarding new customers). Signing stays a no-op until the `TRUSTED_SIGNING_CERT_PROFILE` variable is set (pending Microsoft identity validation of 3D FRANCE).
- The "created" toast now says **TigerCloud**, not "TigerTag Cloud" — the tier's one true name, matching the stats badge (`statCloud`). `TigerCloud` is not a chip: it's a spool tracked with no tag at all — `productCloudCreated` (9 locales)

## Fixed
- Add-product panel: picking a bicolor/tricolor/rainbow colour mode now updates the visible **Aspect 2** dropdown in real time, not just the NFC-data preview — the app-styled dropdown's button label was never refreshed after the value was set programmatically (`_adpSetColorMode` now calls `_cselRefresh`; same fix applied to the Aspect 1 basic↔advanced mirror) — `renderer/inventory.js`
- Add-product panel: the "None" sentinel material (id 65535) is no longer offered in the material picker (sheet + hidden `<select>`) — a real product always has a material — `renderer/inventory.js`
- macOS: closing the window with the red button now hides it (keeping the app and its whole state — Firebase session, inventory, live cameras — alive) instead of destroying it; a Dock-click brings it straight back, with no reload and no re-login. Previously the window was destroyed and the recreated one stayed invisible (a one-shot reveal latch was never reset), so the Dock icon did nothing — `main.js` (close-to-hide on darwin + `_isQuitting` guard, `activate` re-shows, `createWindow` resets `_mainRevealed` as a safety net)
- Add-product panel: the Material selector now shows **PLA** (the actual default) on open instead of a stale **ABS**. The trigger label was synced before the default value was applied, so it echoed the hidden `<select>`'s first alphabetical option (ABS) while the value was already PLA — `renderer/inventory.js` (`openAddProductPanel`)
- Opening the Friends panel now closes every other side card (spool detail, group deck, product card, printer, settings) instead of leaving them stacked underneath — `renderer/inventory.js` (`openFriends` calls `_closeAllSidePanels`)
- `_closeAllSidePanels` was missing the Product card, so it also lingered when opening Notifications — `renderer/inventory.js`
- Reorder ("to order") view: the ✕ that removed a line **deleted the whole product document** — wiping its favorite ★/❤, buy link, price, SKU/EAN and min-stock, not just the order line. Replaced by the non-destructive "saved for later" move (see Added); nothing in that view deletes a product anymore — `renderer/inventory.js`
