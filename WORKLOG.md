# Worklog — v1.8.17 (in progress)

## Added
- Spool detail side panel — new **Balance** mode in the inline weight editor for users with a kitchen scale. A Net / Balance pill toggle sits to the right of the ✓/✕ buttons; in Balance mode the typed value is the scale reading (filament + container) and Studio writes `weight_available = balance − container_weight` to Firestore (reuses the existing `doWeightUpdate(r, "raw", val)` path). The Balance pill's `title` attribute shows the live conversion math ("= 736 g net (contenant : 165 g)") on hover and updates as the user types. Mode persists in localStorage. Input is hard-clamped to capacity (Net) or capacity + container (Balance) on every keystroke + at confirm, so an out-of-range value can no longer reach Firestore — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`.

## Changed
- Spool detail side panel — weight save confirmation. Replaced the verbose 1 s green toast ("✓ N g disponibles (G g − C g contenant) · jumeau mis à jour") with a small green check SVG that pops to the right of the "Poids" section title and fades out smoothly (~1.4 s, 50 % fade window with a slight retreat scale-down). Triggered from every save path (slider auto-save, inline confirm, balance mode) — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`.
- Spool detail side panel — slider auto-save moved from the `input` event to `change`. Pre-fix, pausing the slider mid-drag for 500 ms (still holding the pointer) would burn a Firestore write every pause; now the write is scheduled only when the user releases the thumb, and re-grabbing the slider inside the 500 ms post-release window cancels the pending request — `renderer/inventory.js`.
- Spool detail side panel — server-authoritative slider on incoming snapshot. If a Firestore snapshot arrives while the user is mid-drag or has a pending debounced write, the patch toggles `disabled` for one tick to force-release the slider, applies the remote value to the slider/display/fill, and cancels the local pending write. The user can re-grab to edit again — `renderer/inventory.js`.
- Spool detail side panel — container card. The container brand+model title is now inside the `.cc-meta` column alongside "Customizable" and the weight in grams, beside the container image, instead of sitting on its own line above the card body — `renderer/inventory.js`, `renderer/css/70-detail-misc.css`.

## Fixed
- Spool detail side panel — weight-only patch path was leaving the "Updated" timestamp frozen, silently overwriting a value the user was typing in the manual-edit input, and snapping the slider thumb out from under a mid-drag gesture. The path now refreshes the "Updated" row in place, skips the inline-input write while the manual editor is open, and refuses to override the slider while the user has the pointer down. Also clamped the weight-bar fill percentage to 0-100 in both the friend-view and owner-view markup so a path-2 patch and a path-3 full rebuild produce identical widths when `weight_available > capacity` — `renderer/inventory.js`.
- Spool detail side panel — saving a weight no longer reloads the product image and TigerTag SVG badges. `doWeightUpdate` was forcing a full `openDetail()` rebuild 500 ms after every commit (leftover from before the surgical-patch refactor); removed, since `refreshOpenDetail()` already patches the slider/display/fill/"Updated" row in place when `onSnapshot` echoes the new weight back — `renderer/inventory.js`.
- Spool detail side panel — the "X g — hors plage [0–Y g]" error toast that had no auto-dismiss can no longer fire: typing in the inline editor is hard-clamped on every `input` event (Net max = capacity, Balance max = capacity + container), the confirm path re-clamps as belt-and-suspenders, and `doWeightUpdate` silently clamps as a final safety net. `#panelWeightResult` is also wiped on `openInlineEdit` so any pre-existing stuck toast vanishes — `renderer/inventory.js`.
