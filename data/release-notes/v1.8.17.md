### Added

- **New "Balance" weight input for kitchen-scale users.** In the spool detail panel, the pencil next to the weight now has a Net / Balance toggle. In Balance mode you type the value your scale shows (filament plus container) and Studio subtracts the container weight automatically, so there's no mental math. Hovering the Balance pill shows the live conversion, and the input is capped so you can never type more than the spool's capacity.

### Changed

- Saving a weight no longer reloads the side panel; the value updates in place and the rest of the panel stays put.
- The verbose green save confirmation is gone, replaced by a small check that pops next to the "Weight" title and fades out.
- The weight slider now waits until you release it before saving to the cloud, so a mid-drag pause no longer burns a write; re-grabbing within half a second cancels the pending save.
- If someone else edits the same spool from your phone or another device while you're dragging the slider, the slider now releases and snaps to the value that just arrived, so the display, fill bar and thumb all agree.
- The container name now sits in the same row as its "Customizable" note and gram weight, beside the container thumbnail.

### Fixed

- Fixed an issue where an "out of range" error could get stuck under the weight bar with no auto-dismiss; the input is now clamped at the spool's maximum as you type.
- Fixed an issue where a remote edit to the same spool silently overwrote what you were typing in the manual weight field; your typed value is now kept until you confirm or cancel.
- Fixed an issue where a remote weight update arriving mid-drag made the slider thumb jump out from under your finger.
- Fixed an issue where the "Updated" date in the side panel showed an old timestamp until the next unrelated change; it now refreshes after a weight save.
