### Added

- **Tags.** Label your spools with free-form tags. Add or remove them from a spool's Details panel with autocomplete from tags you already use, filter the inventory with the new All tags dropdown, and find them from the search bar. A spool's tags stay in sync across both chips of a twin pair.
- **Grid view sorting.** The grid now has its own sort menu (brand, material, name, type, weight, capacity, updated) with an ascending/descending toggle, matching the table view.
- **Private chip history.** The app now keeps a private list of every physical chip you've programmed and backs up the repairable signature of each TigerTag+ the first time it is read.

### Changed

- The notifications panel no longer blocks the rest of the app: you can keep clicking around while it is open, close it with a chevron, and opening it tidies away any other open side panel.
- A grouped set of spools in the grid now looks like a normal card with its ×N count badge, without the stacked-paper effect.
- Clearer wording when pairing chips: "Link a second RFID chip" instead of "Link to a twin spool".
- **Firestore explorer (admin).** Debug-enabled accounts get a dedicated tool to browse their own Firestore data with breadcrumb navigation, clickable drill-down, readable values and a raw-JSON view.

### Fixed

- Fixed an issue where "Show in Storage" filled the search bar instead of simply highlighting the spool's slot; it now highlights the slot, and clicking anywhere clears the highlight.
- Fixed an issue where the chip history was not built for accounts whose inventory loaded from cache.
