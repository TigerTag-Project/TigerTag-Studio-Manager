### Added

- **Bambu Lab: full machine controls in the printer panel.** The Bambu side-card now has the same live controls as the other brands: pause / resume / stop the current print, jog the X/Y/Z axes, home any axis, toggle the chamber light, disable the motors, set nozzle and bed targets (preheat while idle), choose the print-speed level (Silent / Standard / Sport / Extreme), and control the part-cooling, auxiliary "assist" and chamber ("Case") fans on enclosed models such as the X1C. Pause / resume / stop use Bambu's documented commands; the other controls rely on community-documented commands and may behave differently on some models.
- **FlashForge Creator 5 & Creator 5 Pro support.** Both models are now recognised. Network discovery and "Add by IP" read the printer's serial number automatically, and the exact model is detected on first connection so the correct name and picture appear without picking them by hand. The Creator 5 Pro is shown as a tool-changer, each tool (T1–T4) with its own filament slot and hotend temperature, and assigning filament to a slot works. Tested on real hardware.

### Changed

- **Bambu Lab: smoother, flicker-free live cards.** The printer card no longer rebuilds itself on every update — only the values that changed are refreshed. Editing a field is no longer interrupted when new data arrives, and the light / fan / speed buttons react instantly.
- **Printer settings open beside the printer card.** Opening a printer's settings now slides the form in next to the printer panel instead of hidden behind it, without dimming the rest of the app, with the same close tab as the other side-cards. Switching to another printer closes a leftover settings form.

### Fixed

- Fixed an issue where Bambu Lab AMS humidity and temperature showed made-up values on the A1's AMS Lite (which has no sensor), and where humidity showed the internal 1–5 dryness grade rather than a real percentage on units that do have a sensor (AMS HT, AMS 2 Pro). Temperature is now shown as a whole number.
- Fixed an issue where newer FlashForge models (e.g. Creator 5) were dropped during discovery when they didn't reveal their identity to the first probe; they now appear in the scan and can be set up via "Add by IP".
- Fixed an issue where the FlashForge Creator 5 Pro's filament slots disappeared and the enclosure ("Case") temperature used the wrong icon; also fixed the Creator 5 Pro catalog image.

> ℹ️ FlashForge Creator 5 / 5 Pro filament colour follows the printer's built-in palette: a colour outside the manufacturer's official set is rejected by the firmware. This is a printer-side constraint, not an app issue.
